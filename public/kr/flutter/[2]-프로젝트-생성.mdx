# flutter create flutter my_app
Flutter 프로젝트를 실행하려면 아래 명령어를 입력하면 됩니다.

```bash:Flutter-프로젝트-생성
flutter create my_app
```

근데 해당 명령어로 생성된 프로젝트 디렉토리(폴더) 에 들어가면 꽤 많은 폴더들이 생성된걸 볼 수 있습니다. Flutter 소개때 말했듯 Flutter 는 크로스 플랫폼을 지원하기 때문에 안드로이드, IOS, 웹, 데스크탑 등등 다양한 플랫폼을 지원하기 위해서 각 플랫폼에 맞는 폴더들이 생성됩니다. 하지만 우린 Android 와 IOS 에 집중할것이기 때문에 이런 사용되지 않는 디렉토리들은 불필요 합니다. 그땐 아래 명령어를

```bash:Android,IOS-프로젝트-생성
flutter create --platforms ios,android my_app
```

위 명령어로 실행하고 난 후 에디터를 열면 이전보다 깔끔하게 생성된걸 볼 수 있습니다. 만약 마음이 바뀌어 나중에 Web 이나 데스크탑을 지원하고 싶다면 그냥 해당 폴더 위치에서 `flutter create .` 명령어를 입력하면 됩니다. 그러면 초기에 생성한 것과 같이 여러 플랫폼을 지원하는 폴더들이 생성됩니다.

# vscode 세팅

흔히 앱개발을 사용할때 Xcode나 Android Studio 를 많이 사용합니다. 하지만 Flutter 는 vscode 를 추천합니다. Flutter 를 만든 팀이 제작한 vscode Extencion 을 이용하면 IDE 만큼 편리하게 사용할 수 있는 여러 강력한 기능들을 제공하기에 추천해요.

먼어 Extencion 에서 Flutter 를 검색하고 두개의 Extencion 을 설치합니다.

그리고 아래의 옵션들을 하나하나씩 설정해 줍니다.

설정 -> 세팅 -> code actions on save 검색 후 settings.json 이렇게 수정

```json:settings.json
"editor.codeActionsOnSave": {
  "source.fixAll": "explicit",
  "source.organizeImports": "explicit",
  "source.sort.json": "explicit",
  "source.sortMembers": "explicit"
},
```

1. 설정 -> 세팅 -> Preview flutter ui guides 체크

2. 설정 -> 세팅 -> format on save 체크

3. 설정 -> 세팅 -> format on type 체크

Flutter 의 대표적인 단점이 코드의 들여쓰기가 많아서 가독성이 괴랄해 진다는건데, 이런 문제를 vscode 가 조금이나마 해결해 줄 수 있습니다. 하지만 이건 개인차가 있으니 본인이 편한대로 설정하시면 됩니다. 전 이렇게 설정했습니다.

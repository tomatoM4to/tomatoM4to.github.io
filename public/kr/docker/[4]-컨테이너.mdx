# 명령어 구조

컨테이너 를 다루기 위한 명령어의 기본 구조는 다음과 같습니다.

```bash
docker container {커멘드} {옵션} {인자}
```

먼저 하나씩 살펴 보기 전 전체적으로 어떤 커멘드가 있는지 확인해 봅시다.

| 커맨드   | 설명                        | 주요 옵션                |
|----------|-----------------------------|-------------------------|
| start    | 컨테이너 실행               | -i                      |
| stop     | 컨테이너 중지               |                         |
| create   | 컨테이너 생성               | --name -e -p -v         |
| run      | 이미지 다운 및 컨테이너 실행 | --name -e -p -v -d -i -t|
| rm       | 컨테이너 삭제               | -f -v                   |
| exec     | 컨테이너 내 명령어 실행      | -i -t                   |
| ls       | 컨테이너 목록 조회           | -a                      |
| cp       | 컨테이너와 호스트 간 파일 복사|                         |
| commit   | 컨테이너를 이미지로 저장     |                         |


***
# 실습
먼저 Docker Desktop 을 실행하고, 커멘드 라인을 열어주세요.

```bash
docker container --help
```

해당 명령어를 윕력하면 commands 에 어떤 명령을 줄 수 있는지 확인할 수 있습니다.

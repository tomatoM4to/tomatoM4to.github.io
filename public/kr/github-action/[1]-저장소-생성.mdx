# first.yml

Action 탭으로 이동하여 `set up a workflow yourself` 를 클릭합니다.

그러면 `gha-tutorial/.github/workflows/main.yml` 파일을 생성하기 위한 에디터가 열립니다.

> `.github/workflows` 디렉토리 내부에 `.yml` 확장자를 가진 파일이 존재하면 GitHub Actions 이 활성화됩니다.

이때 잠시 `main.yml` 파일 이름만 `first.yml` 로 이름을 변경합니다.

그 후 아래의 코드를 입력합니다.

```yaml
name: First Workflow
on: push
jobs:
  example_job:
    runs-on: ubuntu-latest
    steps:
      - run: echo 'Hello, Github Actions!'
  pwd-ls:
    name: PWD & LS
    runs-on: ubuntu-latest
    steps:
      - run: pwd
      - run: ls -al
```

* `name: First Workflow` : 워크플로우의 이름을 지정합니다. 필수 항목은 아닙니다.
* `on: push` : 워크플로우가 실행되는 이벤트를 지정합니다. **이 항목은 필수입니다.** 여기선 가장 많이 사용되는 `push` 이벤트를 사용합니다.
* `jobs` : 워크플로우에서 실행할 작업들을 정의합니다. 각 작업은 **job_id** 로 식별됩니다.
* `example_job` : **첫 번째 job_id** 입니다.
* `runs-on: ubuntu-latest` : 이 작업이 실행될 환경을 지정합니다. 여기선 최신 버전의 Ubuntu를 사용합니다. 이밖에도 `windows-latest`, `macos-latest` 등을 사용할 수 있습니다.
* `steps` : 작업에서 실행할 단계들을 정의합니다. 각 단계는 순차적으로 실행됩니다.
* `- run: echo 'Hello, Github Actions!'` : 이 단계에서 실행할 명령어를 지정합니다. 여기선 `echo` 명령어를 사용하여 "Hello, Github Actions!" 메시지를 출력합니다.
* `pwd-ls` : **두 번째 job_id** 입니다. 이 작업은 현재 작업 디렉토리의 경로를 출력하고, 그 디렉토리 내의 파일 목록을 나열합니다.
* `name: PWD & LS` : 이 작업의 이름을 지정합니다. 역시 필수 항목은 아닙니다.
* `- run: pwd` : 현재 작업 디렉토리의 경로를 출력합니다.
* `- run: ls -al` : 현재 작업 디렉토리 내의 파일 목록을 자세히 출력합니다.

> `runs-on` 항목은 워크플로우가 실행될 환경을 지정합니다. GitHub Actions는 다양한 운영 체제를 지원하며, 각 운영 체제에 따라 사용할 수 있는 기능이 다를 수 있습니다.

이후 우측 상단의 **Commit changes...** 버튼을 클릭하여 변경 사항을 커밋합니다.

다시 Action 탭으로 이동하면 방금 작성한 워크플로우가 실행되는 것을 확인할 수 있습니다.

각 job 의 실행 결과를 확인할 수 있으며, 클릭하면 각 단계의 실행 로그를 볼 수 있습니다.

실행 로그는 기본적으로 다음과 같이 표시됩니다.

* Set up job : 작업 환경을 설정합니다.
* Run `step_name` : 각 단계에서 실행된 명령어를 표시합니다.
* Complete job : 작업이 완료되었음을 표시합니다.

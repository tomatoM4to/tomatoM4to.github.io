What is Null?

Null Safety를 알아보기 전에 먼저 null이라는 개념부터 간단히 정리해보겠습니다. null은 대부분의 프로그래밍 언어에서 공통적으로 사용하는 개념이기 때문에 익숙하실 거예요.

간단히 말해, null은 "값이 없는 상태"를 의미합니다. 예를 들어, Dart나 Java 같은 언어에서 변수는 메모리 주소를 참조하게 되는데, 이 주소에 유효한 Object가 존재할 수도 있고, 반대로 아무것도 참조하지 않는 null 상태일 수도 있습니다.

즉, 변수는

  • 어떤 유효한 값을 참조
  • 아무것도 참조하지 않는 null 상태

두 가지 특성을 가질 수 있는 것이죠.

실제로는 파일을 읽거나, 데이터베이스에서 값을 조회할 때, 값이 존재하지 않으면 null을 반환하는 방식으로 자주 사용됩니다.

물론 상황에 따라 true/false, try/catch, 혹은 -1 같은 특정 상수값을 사용하는 방식으로 null을 대체하기도 합니다.

하지만 Dart에서는 이러한 상황에서 null을 직접 사용하는 경우가 꽤 많고, 이를 안정적으로 처리하기 위해 Null Safety라는 개념이 도입되었습니다.

어떤 입장에선 런타임 에러가 발생하지 않아서 다행이라고 느낄 수도 있겠지만, 만약 이런 예상치 못한 값이 중요한 서비스에 영향을 준다면? 상상만 해도 아찔하죠.

그래서 결국 개발자가 직접 null 체크를 해줘야 하는 상황이 자주 발생합니다.

물론 이런 방식 덕분에 런타임 전에 오류를 미리 잡을 수 있다는 장점은 있지만요.

Null Safety

Null Safety 를 간단히 말하면, 변수가 null 값을 참조하지 못하도록 제한하는 기능입니다.

가장 중요한 점은, Dart의 Null Safety 는 런타임이 아니라 컴파일 타임, 즉 코드를 작성하는 시점에 null 가능성을 미리 검사한다는 거예요.

이 덕분에 앱을 실행하기도 전에 null 과 관련된 오류를 IDE 에서 미리 확인하고 수정할 수 있습니다. 결과적으로 예기치 않은 런타임 크래시를 방지할 수 있게 되는 거죠.

Dart 공식 문서에서는 Null Safety 를 이렇게 설명합니다:

Sound null safety changes potential runtime errors into edit-time analysis errors.


번역: 널 세이프티는 잠재적인 런타임 오류를, 코드 작성 시점의 분석 오류로 바꿔줍니다.


https://dart.dev/null-safety

여기서 말하는 Sound null safety 란, 모든 변수 타입이 기본적으로 null 을 허용하지 않도록 하고, 정적 분석과 컴파일러 최적화를 통해 런타임에서 null 관련 오류가 절대 발생하지 않도록 보장하는 것을 말해요.

일반적인 null safety 와 개념적으로 큰 차이는 없지만, Dart 에서는 이 점을 더 명확하게 하기 위해 Sound null safety 라는 용어를 사용하고 있습니다.

Null 관련 Keyword 정리

Dart 의 핵심 기능 중 하나인 Null Safety 덕분에, 관련 Keyword 도 여러 개 존재합니다. 하나씩 간단히 살펴볼게요.

Keyword설명
?nullable 타입 선언
!null 아님을 강제 (non-null 단언)
??null일 경우 대체값 사용
late나중에 초기화할 non-null 변수

? - nullable 타입 선언

null 이 될 수 있는 변수를 선언할 때 사용합니다. nullable 로 선언된 변수는 사용 전에 항상 null 체크가 필요합니다.

null-Keyword

! - null 아님을 강제 (non-null assertion)

컴파일러에게 이 값은 절대 null 이 아님 이라고 단정할 때 사용합니다. 하지만 실제로 null 이면 런타임 에러가 발생하므로 주의해야 해요.

null-Keyword

?? - null 일 경우 대체값 사용

왼쪽 값이 null일 경우, 오른쪽 값을 대신 사용합니다. 자주 쓰이는 패턴이라 꼭 익혀두면 좋아요.

null-Keyword

late - 나중에 초기화할 non-null 변수

null 이 될 수 없는 변수지만, 선언 시에 바로 초기화하기 어려운 경우 사용합니다. 예를 들어 생성자나 외부 로직을 통해 나중에 값이 설정될 경우에 유용합니다.

null-Keyword

?. — null-safe 접근 연산자

객체가 null 이 아닐 경우에만 해당 속성이나 메서드에 접근하도록 합니다.

What is Null?

Null Safety를 알아보기 전에, 먼저 null 이라는 개념부터 정리해보겠습니다. null 은 많은 프로그래밍 언어에서 공통적으로 사용되는 개념이기 때문에 대부분 익숙하실 거라 생각합니다.

간단히 말해, Null은 값이 없는 상태를 의미합니다. 예를 들어, Dart 나 Java 같은 언어에서 모든 변수는 어떤 주소값을 가지게 되는데, 이 주소에 실제로 유효한 Object가 존재할 수도 있고, 반대로 아무것도 없는 null 상태일 수도 있습니다.

즉, 변수는 유효한 데이터를 참조하거나, 혹은 아무것도 참조하지 않는 상태인 null를 가질 수 있다는 두 가지 특성을 지니고 있는 셈이죠.

보통 파일을 읽거나 데이터베이스에서 특정 데이터를 조회할 때, 해당 데이터나 파일이 존재하지 않는 경우 null 값을 반환하는 방식으로 자주 사용됩니다.

물론 상황에 따라 true/false 값을 사용하거나, try/catch 로 예외를 처리하거나, 혹은 -1 같은 특정 상수를 반환하는 방식으로 null을 대신하기도 합니다.

하지만 Dart에서는 이러한 상황에서 null을 직접 사용하는 경우가 매우 많습니다.

Null 로 인한 문제점

Null Safety를 본격적으로 살펴보기 전에, 먼저 Null Safety를 보장하지 않는 대표적인 언어, 바로 JavaScript의 예시부터 알아보겠습니다.

main.js
const add = (a, b) => a + b;

console.log(add(5, 3));     // 8
console.log(add(5, null));  // 5
console.log(add(null, null)); // 0
console.log(add(5));        // NaN

add 함수는 단순히 두 값을 더하는 한 줄짜리 화살표 함수입니다. 하지만 JavaScript는 타입 검사나 null에 대한 제약이 없기 때문에, null이나 undefined 같은 값이 들어가도 에러 없이 실행되며, 때로는 예상하지 못한 결과(0, NaN)를 낳을 수 있습니다.

어떻게 보면 JavaScript의 동작 방식이 다행스럽다고 느낄 수도 있습니다. 런타임 에러가 발생해 앱이 바로 종료하지는 않으니까. 만약 실행중인 서버나 앱이 이런식으로 종료된다면... 상상만 해도 아찔한 상황이겠죠.

이런 문제를 피하려면 결국 개발자가 직접 null 체크를 해줘야 합니다.

main.js
const add = (a, b) => {
    if (a === null || b === null) {
        return throw new Error('Invalid argument: null value');
    }

    // null safety 한 영역
    return a + b;
};

console.log(add(5, 3));     // 8
console.log(add(5, null));  // Error: Invalid argument: null value
console.log(add(null, null)); // Error: Invalid argument: null value
console.log(add(5));        // Error: Invalid argument: null value

이렇게 작성하면 null 값이 들어오는 상황을 사전에 방지할 수 있습니다.

하지만 코드가 길어지고, 가독성도 떨어지며, 매번 이런 식의 방어 코드를 작성해야 한다는 점은 번거로운 일입니다. 그래도 이 방식 덕분에 프로그램을 배포하기 전 오류를 미리 발견할 수 있다는 장점은 있습니다.

그렇다면 이제, Dart 에서 이러한 문제들을 어떻게 해결하고 있는는지 살펴보겠습니다.

Null Safety

Null Safety를 간단하게 말하자면, 변수가 null 값을 참조할 수 없도록 제한한 하는 기능입니다.

가장 중요한 점은 Dart 의 Null Safety가 런타임이 아닌 컴파일 타임, 즉 코드를 작성하는 시점에 null 가능성을 미리 검사한다는 점입니다.

이로 인해 코드에서 발생할 수 있는 null 관련 오류를 앱이 실행되기도 전에 미리 확인하고 수정할 수 있게 됩니다.

Dart에서는 이를 다음과 같이 표현합니다.

Sound null safety changes potential runtime errors into edit-time analysis errors.


번역: 널 세이프티는 잠재적인 런타임 오류를, 코드 작성 시점의 분석 오류로 바꿔줍니다.


https://dart.dev/null-safety

여기서 Sound null safety란, 코드의 모든 타입을 기본적으로 null이 될 수 없도록 만들고, 정적 분석과 컴파일러 최적화를 통해 런타임에서 널 참조 오류가 발생하지 않도록 보장하는 개념입니다.

사실상 일반적인 Null safety와 기능적으로 큰 차이는 없지만, Dart에서는 이를 Sound null safety라는 이름으로 명확하게 표현하고 있습니다.

main.dart
bool isEmpty(String s) {
    return s.length == 0;
}

void main() {
    isEmpty(null); // null 값을 전달
}
result
Error: The value 'null' can't be assigned to the parameter type 'String' because 'String' is not nullable.

IDE에서도 이 부분에 빨간 줄이 그어지며 에러를 미리 알려주기 때문에, 런타임에서 갑자기 앱이 종료되는 최악의 상황을 방지할 수 있습니다.

Null able

하지만 가끔 null 값을 허용해야 하는 경우도 있습니다. 이럴땐 아주 간단하게 ? 를 붙여주면 됩니다.

main.dart
void main() {
    String? name = null; // null 값을 허용하는 String 타입
    print(name); // null

    name = 'Dart';
    print(name); // Dart
}

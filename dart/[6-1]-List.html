Collections

Collections 는 Dart 에서 여러 값을 한꺼번에 담을 수 있는 자료구조를 의미합니다. 한국어로는 흔히 반복 가능한 자료구조 또는 컬렉션 타입이라고 부르죠.

Dart 의 주요 컬렉션 타입은 아래 세 가지입니다.

타입설명
List순서가 있는 데이터 집합 (배열과 유사)
Set중복을 허용하지 않는 집합
Map키와 값을 쌍으로 저장하는 구조

이들은 모두 Iterable 이라는 인터페이스를 따르며, for 문이나 forEach 등을 통해 반복(iterable) 할 수 있습니다.

이러한 세 가지 타입은 Dart 에서 모두 컬렉션(Collection) 이라고 부릅니다.

List

Dart에서 List는 여러 값을 순서대로 저장할 수 있는 컬렉션 타입입니다. List 는 제네릭 타입이라, 어떤 타입의 값들을 저장할지 명시할 수 있어요.

list-선언

이런 방식 외에도 생성자를 이용해 만들 수 있습니다.

list-생성자

Dart 에서는 리스트나 세트, 맵 같은 컬렉션 리터럴 안에서 if 와 for 을 사용할 수 있습니다. 이를 각각 Collection if, Collection for라고 부릅니다.

이 기능을 활용하면 더 간결하고 읽기 쉬운 코드로 조건에 따라 요소를 추가하거나, 반복적으로 요소를 생성할 수 있습니다.

Loop

List 에서 각 요소를 반복하기 위해선 4가지 방법이 있습니다. for-in 부터 차례대로 살펴보겠습니다.

방법설명
for전통적인 인덱스 기반 반복문
for-in요소 하나씩 꺼내며 반복
forEach각 요소에 대해 콜백 함수 실행
map각 요소를 변형해 새로운 컬렉션 생성

for-in

for-in 문은 List 의 요소를 하나씩 순회할 때 가장 자주 사용되는 방법입니다.

list-for-in

변수를 수정하지 않는다면 final 을 붙여주는 것이 좋습니다. 물론 생략해도 전혀 문제는 없어요.

forEach

forEach() 는 List 의 각 요소에 대해 콜백 함수를 실행합니다. 함수형 프로그래밍 스타일을 좋아하는 사람들에게 익숙한 방식이에요.

list-forEach

forEach() 는 메서드이기 때문에 반환값이 존재하며, 그 값은 void 입니다. 결과를 수집할 수는 없고, 단순 반복에 적합합니다.

map

많은 분들이 forEach 와 map 을 헷갈려 하시는데, 가장 큰 차이점은 map 은 새로운 값을 반환한다는 것입니다!

list-map

map() 의 반환값은 Iterable 타입입니다. 즉, 리스트처럼 반복할 수는 있지만, 아직 완전한 List는 아니에요.

리스트로 바꾸려면 .toList() 를 붙여줘야 합니다.

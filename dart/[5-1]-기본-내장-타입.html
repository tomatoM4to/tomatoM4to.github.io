Built-in types

기본 내장 타입 혹은 Built-in types 는 Dart 에서 제공하는 기본 데이터 타입니다. 쉽게말해 그냥 int, double, String 등과 같은 타입을 말합니다. 먼저 직관적으로 바로 이해할수 있는 타입들부터 살펴보겠습니다.

  1. int : 정수형 타입
  2. double : 실수형 타입
  3. String : 문자열 타입
  4. bool : 불리언 타입 (true/false)
  5. List : 리스트 타입 (배열과 비슷)
  6. Map : 맵 타입 (key-value 쌍으로 이루어진 데이터 구조)
  7. Set : 집합 타입 (중복을 허용하지 않는 데이터 구조)
  8. Null : null 타입 (값이 없음을 나타내는 타입)

이 밖에도 Runes, Symbol, Records, Function 같은 타입도 존재하지만, 자주 사용하지도 않고, 한번에 설명하기엔 햇갈리기만 하기 떄문에 나중에 자세히 다루겠습니다.

중요한 점은 이러한 방식의 타입 선언엔 객체를 생성하는 능력을 포합합니다!!!

Dart 공식문서에서 이러한 설명이 존재합니다.

Uninitialized variables that have a nullable type have an initial value of null. Even variables with numeric types are initially null, because numbers—like everything else in Dart—are objects.


https://dart.dev/language/variables

요약하면 다음과 같습니다. Dart 는 숫자 조차도 Object로 취급한다. 즉, int 나 double 타입도 Object로 취급되며, 이러한 객체들은 메모리에서 공간을 차지합니다. 따라서 Dart 에서 제공하는 기본 내장 타입은 모두 Object입니다.

결국 int i = 0; 이란 간단한 코드는 내부적이로로 이렇게 동작하게 됩니다.

  1. 정수 1 은 int 타입의 객체로 힙(Heap) 메모리에 저장
  2. 변수 i 는 int 타입의 객체(Object) 를 참조하는 포인터(Reference) 역할을 함

Numbers

Dart 에서 숫자를 다루는 방식은 다른 언어와 크게 다르지 않습니다. Dart 는 기본적으로 int 와 double 타입을 제공합니다. 그리고 잘 사용되진 않지만 num 이라는 타입도 있습니다. num은 int 와 double 의 부모 타입이라고 생각하시면 됩니다.

또한, 큰 숫자를 더 읽기 쉽게 표현하기 위해 언더스코어를 사용할 수 있는 점도 Dart의 특징 중 하나입니다.

int

플랫폼에 따라 다를 수 있지만, 네이티브 플랫폼에서는 정수 가 표현할 수 있는 값의 범위는 −263-2^{63} 부터 263−12^{63}-1 사이 입니다.

double

IEEE 754 표준에 따른 64비트(배정밀도) 부동 소수점 숫자를 표현합니다.

num

num 은 int 와 double 을 모두 포함하는 타입입니다. 즉, 이 두 타입의 부모 타입이며, 일반적인 사칙연산을 포함해 abs() 같은 메서드도 여기에 포함되어 있습니다.

int 가 객체라고?

다른 언어에서는 객체와 기본 데이터 타입(예: 정수, 실수, 문자)을 구분하는 경우가 많습니다.

하지만 Dart 에서는 숫자든 문자든 모두 객체입니다. 따라서 다음과 같은 것도 가능합니다:

객체성

Dart 공식 문서에서도 이렇게 설명하고 있습니다:

n Dart, all numbers are part of the common Object type hierarchy, and there are two concrete, user-visible numeric types: int, representing integer values, and double, representing fractional values.


https://dart.dev/resources/language/number-representation

하지만 걱정할 필욘 없습니다. 숫자의 객체성으로 인해 다른 함수로 인자를 전달할때 call by reperence 로 전달되는거 아닌가? 하는 그런 걱정은 할필욘 없습니다. Dart 에선 숫자도 객체이지만, 숫자에 대한 연산은 call by value 처럼, 정확히는 ...

Call by Reference?

숫자도 객체라면, 함수에 인자로 전달할 때 Call by Reference 로 동작하는 거 아닌가? 라는 의문이 생길 수 있습니다.

하지만 걱정할 필요 없습니다. Dart 에서는 int, double, bool, String 등 기본 타입은 모두 immutable objects로 설계되어 있기 때문에, 마치 Call by Value 처럼 동작하거든요.

정확히 말하면, 변수는 객체의 reference 를 담고 있는 것이 맞습니다. 하지만 중요한 건 그 객체가 수정 가능한(mutable)이냐, 수정 불가능한(immutable)이냐는 점이에요.

example

위 코드를 보면 b = a 를 했을 때 b 가 a 의 값을 그대로 가져온 것처럼 보이죠. 하지만 실제로는 a 가 참조하고 있는 10 이라는 int 객체를 b도 함께 참조한 겁니다. 그래서 identical(a, b) 는 true 를 반환합니다.

  • identical 는 두 객체 reference 가 동일한 객체에 대한 reference 인지 확인하는 함수 입니다.

그런데 int는 불변 객체이기 때문에, b = 20을 하면 기존 객체를 바꾸는 게 아니라, b가 새로운 int 객체 20을 참조하게 되는 거예요. 즉, a는 여전히 10을 가리키고 있기 때문에 값이 변하지 않은 겁니다.

이러한 immutable 한 특성을 가진 대표적인 타입은 int, double, String, bool 등이 있습니다.

Strings

Dart의 문자열은 String 타입 객체이며, 내부적으로 UTF-16 코드 유닛의 연속된 배열(시퀸스)로 구성되어 있습니다. 문자열을 만들 때는 Python 처럼 작은 따옴표(') 나 큰 따옴표(") 모두 사용할 수 있어요.

프로그래밍 언어가 문자를 어떻게 다루는지에 대한 설명은 자세히 하지 않겠습니다. 그냥 간단히 말해서, Dart는 문자를 저장할 때 UTF-16 인코딩을 사용한다고 생각하면 됩니다.

Booleans

Dart 에서 boolean 값을 표현할 땐 bool 타입을 사용합니다. 이 타입에는 오직 두 가지 값, true와 false만 존재합니다.

여기서 꼭 기억해야 할 점은, Dart는 타입에 엄격한 언어이기 때문에 if (1)이나 if ('문자열')처럼 bool 이 아닌 값은 조건식에 쓸 수 없습니다!

즉, 숫자나 문자열의 truthy/falsy 개념은 Dart에선 동작하지 않아요. 조건을 쓸 땐 항상 ==, .isEmpty, .isNaN 같은 명시적인 방식으로 판단해야 합니다.

# Introduction
해당 주제는 https://ruslanspivak.com/lsbaws-part1/ 를 정리한 주제입니다. 내용이 마음에 들었다면 원 저자의 글을 읽어보시는 것을 추천합니다.

해당 글의 저자는 이렇게 말합니다.

> 훌륭한 개발자가 되기 위해선 매일 사용하는 소프트웨어 시스템의 근본적인 원리를 깊이 이해해야 한다 믿습니다. 여기에는 프로그래밍 언어, 컴파일러와 인터프리터, 데이터베이스와 운영체제, 웹 서버와 웹 프레임워크가 포함됩니다. 그리고 이러한 시스템을 깊이 이해하기 위한 최고의 방법은 처음부터 다시 만들어 보는겁니다. 벽돌 하나하나를 쌓고, 벽 하나하나를 만들어가는 과정이 중요합니다.

또 공자는 이렇게 말했다고 합니다.

> 듣고 잊는 것은 알지 못하는 것과 같다. 보고 기억하는 것은 약간 알고 있는 것과 같다. 하지만 직접 해보는 것은 알고 있는 것과 같다.

저또한 직접 해보는것이야 말로 가장 중요하다고 생각합니다. 그래서 여러 문서들을 찾아봤고 이 글을 작성하게 되었습니다.

근데 또 이런 생각도 들더라구요. 예전에 들어본 말인데, 누군진 기억이 나지 않습니다.

> 남의 실수로부터 배우는 사람이 1류, 자신의 실수로부터 배우는 사람은 2류, 아무것도 배우지 않는 사람은 3류다.

저는 제 눈으로 직접 보지 못하면 죽어도 이해가 안되는 사람이라서, 이 글을 작성하게 되었습니다. 이 사람의 말에 따르면 전 아무래도 2류에 속하는 것 같네요.

또 교수님과의 진로 상담을 해본적이 있는데요, 해당 교수님은 극한의 실용주의자 였던거 같습니다. 그래서 확실하게 목표를 정하고 의미있고 새로운 일을 해야한다 강조하셨습니다. 처음엔 지식을 공유하는 나의 낭만적이고 이타적이면서 고귀한 어쩌구를 무시하다니.. 했지만 생각해보면 확실히 제 블로그는 너무 광범위한 경향이 있는것 같습니다. 그리고 결국 인터넷이나 교과서에 있는 글들을 모아모아 정리한것이라는 생각도 들더라구요.

확실히 처음부터 무언갈 만들어보는것은 중요하다 생각합니다. 하지만 그게 시간이 너무 오래걸릴거 같다 생각하시면 그냥 글만 읽어도 좋은 방법이라고 생각합니다. 여러사람의 관점으로 생각해보는건 중요하니까요. 전 그냥 만들어보렵니다.

## 정렬

### 버블 정렬 (Bubble Sort)
총 `n-1` 회전이 필요하며 `O(n^2)` 의 시간 복잡도를 가짐

* 1회전 `5, 1, 7, 10, 2` -> `1, 5, 7, 2, 10` 10 고정
* 2회전 `1, 5, 7, 2, 10` -> `1, 5, 2, 7, 10` 7, 10 고정
* 3회전 `1, 5, 2, 7, 10` -> `1, 2, 5, 7, 10` 5, 7, 10 고정
* 4회전 `1, 2, 5, 7, 10` -> `1, 2, 5, 7, 10` 2, 5, 7, 10 고정

example code
```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n-i-1):
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
    return arr
```

### 삽입 정렬 (Insertion Sort)
좌, 우로 구분하여 좌측은 정렬된 부분, 우측은 정렬되지 않은 부분으로 나누고, 우측에서 하나씩 요소를 선택하여 좌측의 적절한 위치에 삽입하는 방식으로 정렬하는 알고리즘

총 `n-1` 회전이 필요하며 `O(n^2)` 의 시간 복잡도를 가짐

* 1회전 `5, 1, 7, 10, 2` -> `1, 5, 7, 10, 2`, 5와 1을 비교, `1, 5` 를 정렬되었다 고 생각
* 2회전 `1, 5, 7, 10, 2` -> `1, 5, 7, 10, 2`, 5와 7을 비교, `1, 5, 7` 을 정렬되었다 고 생각
* 3회전 `1, 5, 7, 10, 2` -> `1, 5, 7, 10, 2`, 7과 10을 비교, `1, 5, 7, 10` 을 정렬되었다 고 생각
* 4회전 `1, 5, 7, 10, 2` -> `1, 2, 5, 7, 10`, 10과 2를 비교, 7과 2를 비교, 5와 2를 비교하여 2를 1과 5 사이에 삽입

### 선택 정렬 (Selection Sort)
`0 + i` 번째 요소부터 `n-1` 번째 요소까지의 요소 중에서 가장 작은 요소를 선택하여 `i` 번째 요소와 교환하는 방식으로 정렬하는 알고리즘

총 `n-1` 회전이 필요하며 `O(n^2)` 의 시간 복잡도를 가짐

* 1회전 `5, 1, 7, 10, 2` -> `1, 5, 7, 10, 2`, 1이 가장 작은 요소이므로 5와 교환
* 2회전 `1, 5, 7, 10, 2` -> `1, 2, 7, 10, 5`, 2가 가장 작은 요소이므로 5와 교환
* 3회전 `1, 2, 7, 10, 5` -> `1, 2, 5, 10, 7`, 5가 가장 작은 요소이므로 7과 교환
* 4회전 `1, 2, 5, 10, 7` -> `1, 2, 5, 7, 10`, 7이 가장 작은 요소이므로 10과 교환

## Big-O
* $O(n)$ -> $O(n)$
* $O(3n^2+2n+1)$ -> $O(n^2)$
* $O(5)$ -> $O(1)$

정렬 알고리즘 Big-O

| 알고리즘 | 최선 | 평균 | 최악 |
| --- | --- | --- | --- |
| 버블(Bubble) | $O(n)$ | $O(n^2)$ | $O(n^2)$ |
| 삽입(Insertion) | $O(n)$ | $O(n^2)$ | $O(n^2)$ |
| 선택(Selection) | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ |
| 합병(Merge) | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ |
| 퀵(Quick) | $O(n \log n)$ | $O(n \log n)$ | $O(n^2)$ |
| 기수(Radix) | $O(n \cdot k)$ | $O(n \cdot k)$ | $O(n \cdot k)$ |
| 셸(Shell) | $O(n \log n)$ | $O(n^{1.5})$ | $O(n^2)$ |
| 힙(Heap) | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ |

* `n` : 정렬할 항목의 개수
* `k` : 각 항목의 최대 자릿수

## 단위 모듈 테스트 방법

* Black-box : 사용자 관점
* White-box : 개발자 관점

### Black-box Testing
1. 사용자 요구 사항 명세를 보면서 테스트한다.
2. 구현된 기능을 테스트 한다.
3. 해당 기능의 작동 여부를 테스트 한다.

> 동등 분할 검사, 경계값 분석, 오류 예측

> 테스트 쉬움

### White-box Testing
1. 프로그램의 내부 로직 (수행경로 구조, 루프 등)을 보면서 테스트를 수행한다.
2. 소스코드의 논리적인 경로를 테스트 한다.

> 기초 경로 검사, 제어 구조 검사

> 오류를 발견하기 쉬움

## 저작권 보호 기술(DRM)의 흐름
* 콘텐츠 제공자 : 영화 스튜디오, 뉴스 미디어 회사 등
* 콘텐츠 분배자 : 스트리밍 서비스(넷플릭스), 케이블 TV 제공 업체, 인터넷 서비스 제공 업체
* 콘텐츠 소비자 : 시청자, 청취자, 독자
* 클리어링 하우스 : 디지털 저작권 관리(DRM) 시스템, 결제 처리 화사(PayPal, Visa)
* 패키저(Packager) : 콘텐츠를 메타데이터와 함께 배포 가능한 단위로 묶는 기능
* 보안 컨테이너 : 원본을 안전하게 유통 하기 위한 전자적 보안 장치
* DRM 컨트롤러 : 배포된 콘텐츠의 이용 권한을 통제

DRM 의 기술 요소
* 암호화(Encryption) : 콘텐츠 및 라이선스를 암호화하고, 전자 서명을 할 수 있는 기술
* 키 관리(Key Management) : 콘텐츠를 암호화한 키에 대한 저장, 배포, 파기 기술
* 암호화 파일 생성(Packager) : 콘텐츠를 암호화된 콘텐츠로 생성하기 위한 기술
* 식별 기술(Identification) : 객체의 위치와 상관없이 해당 객체를 고유하게 식별하는 "DOI", 웹주소(URL) 와 같은 리로스를 가리키기 위해 사용하는 "URI" 등
* 저작권 표현(Rights Expression) : 라이선스의 내용 표현 기술로 디지털 권리 및 라이선싱 정보를 표현하고 배포
* 정책 관리(Policy Management) : 라이선스 발급 및 사용에 대한 정책 표현 및 관리기술
* 크랙 방지(Tamper Resistance) : 크랙에 의한 콘텐츠 사용 방지 기술로 악의적인 공격자로부터 시스템이나 앱을 보호
* 인증(Authentication) : 라이선스 발급 및 사용의 기준이 되는 사용자 인증 기술

## EAI(Enterprise Application Integration)
* Point-to-Point : 시스템 간 직접 연결, 시스템이 많아질수록 복잡도가 증가
* Hub-and-Spoke : 중앙 허브를 통해 시스템 간 연결, 허브가 단일 실패 지점이 될 수 있음
* Message Bus : 시스템이 메시지 버스(Middleware)를 통해 통신, 확장성 높음, 복잡도 증가
* Hybrid : Point-to-Point, Hub-and-Spoke(그룹 내), Message Bus(그룹 간) 의 조합, 유연성 높음, 관리 복잡도 증가

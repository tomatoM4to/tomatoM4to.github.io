## 소프트웨어 생명주기 모델(SDLC)
* 폭포수 모델(Waterfall Model): 각 단계가 끝난 다음 다음단계로 진행, 가장 오래된 방식
  * `요구사항 분석 -> 설계 -> 구현 -> 테스팅 -> 유지보수`
* 원형 모델(Prototyping Model): Prototype 제품 먼저 개발 후 완제품 개발
  * `요구사항 정의 -> 원형 설계 -> 원형 개발 -> 고객 평가 -> 요구사항 조정 -> 완제품`
* 나선형 모델(Spiral Model): 소프트웨어 개발 위험을 최소화 하기 위해 4단계를 반복하며 점진적으로 완성하는 모델
  * `계획 및 정의 -> 위험 분석 -> 개발 -> 고객 평가`
* V 모델: 소프트웨어 개발의 각 단계에서 대응하는 테스트 단계를 명확히 정의, 폭포수 모델의 변형
  * `요구사항 분석 -> 기능 명세 분석 -> 설계 -> 개발` 이후 개발 단계부터 `단위 테스트 -> 통합테스트 -> 시스템 테스트 -> 인수 테스트`

## 소프트웨어 비용 추정모형(Estimation Models)

### COCOMO 모형
유형별 분류
1. **Organic**: 50,000라인 이하
2. **Semi-Detached**: 300,000 라인 이하
3. **Embedded**: 300,000 라인 초과

형태별 분류
1. **Basic**: **단순**한 모델로 기본적인 비용 산정
2. **Intermediate**: **중간** 단계의 세부 인수를 포함한 비용 산정
3. **Detailed**: **상세**한 단계별 비용 산정

## CMMI 모델의 성숙 단계
조직의 소프트웨어 시스템 개발 프로세스 성숙도 평가 지표

1. 초기(Initial) 단계: 프로세스가 무질서하고 임기응변식으로 이루어진 상태
2. 반복(Managed) 단계: 다소 통계정 관리가 가능한 상태
3. 정의(Defined) 단계: 프로세스가 표준화되고 문서화되어 있는 상태
4. 관리(Quantitatively Managed) 단계: 프로세스 성과 측정, 분석, 개선, 관리가 가능한 상태
5. 최적(Optimizing) 단계: 개선이 지속적으로 진행되는 상태

## SPICE 모델의 성숙 단계
| 레벨 | 성숙 단계 | 설명 |
|---|---|---|
| 레벨 0 | 불완전(Incomplete) | 프로세스가 공식적으로 정의되지 않은 상태 |
| 레벨 1 | 수행(Performed) | 프로세스가 문서화되지 않은 상태 |
| 레벨 2 | 관리(Managed) | 기본적인 관리 활동이 수행되는 상태 |
| 레벨 3 | 확립(Established) | 프로세스가 표준화된 상태 |
| 레벨 4 | 예측(Predictable) | 프로세스의 성과가 예측 가능한 상태 |
| 레벨 5 | 최적화(Optimizing) | 지속적인 개선이 이루어지는 상태 |

## RIP vs OSPF
Network 관련 내용

* RIP: 홉 카운트(hop count)를 기준으로 최적의 경로를 선택하는 거리벡터라우팅 프로토콜, 최대 홉수는 15, 주기는 30초
  * 간단하고 설정이 쉬우며, 작은 네트워크에 적합
2. OSPF: Dijkstra 알고리즘을 사용하여 최단 경로를 계산
  * 복잡하지만 큰 네트워크에 적합하며, 빠른 수렴성과 효율적인 경로를 선택 가능

## 암호 알고리즘 종류

### 대칭키 암호화
암호화, 복호화 키가 같음
* 빠르고 계산이 간단
* 키관리가 어렵다.

> 1. 블록 암호 방식: AES, DES, 3DES, SEED
> 2. 스트링 암호 방식: RC4

* **DES**: **64비트**중 56비트의 키는 암호화 복호화에 사용하고 나머지 8비트는 키블록의 parity check 용으로 사용
* **AES**: DES 가 뚫려서 고안됨, **128비트** 블록 크기를 사용
* **SEED**: **한국 국가 표준** 암호화 알고리즘, 주로 금융과 공공기관에서 사용

### 비대칭키 암호화
암호화와 복호화에 서로 다른키 사용 (공개키, 비밀키)

* **RSA**: 큰 정수를 소인수분해 하는것이 매우 어렵다는 수학적 문제에 기반
* **Diffie-Hellman**: 큰 소수의 제곱을 계산하는 것은 쉽지만, 그 제곱근을 역으로 찾는 것은 매우 어렵다는 문제에서 기반
* **엘가말(ElGamal)**: 큰 정수에 대한 로그 계산이 매우 어렵다는 사실을 이용
* **타원곡선 암호화(ECC)**: RSA 에 비해 훨씬 더 작은 키 크기로 동일한 수준의 보안을 제공

> 비대칭키는 복잡한 수학 공식(소인수분해 등)을 풀기 때문에 대칭키보다 수백 배 느림

### 해시 함수
입력된 데이터를 고정된 크기의 해시 값으로 변환하며, 보통 **단방향**

> SHA-256, MD5

## 서비스 공격
* **스푸핑(Spoofing)**: 공격자가 자신을 다른 사람이나 시스템으로 속이는 방식
* **Teardrop 공격**: 패킷 조각들이 비정상적인 오프셋(offset) 값을 갖도록 하는 공격 기법
* **Smurf 공격**: 네트워크 브로드캐스트 주소를 이용해 대량의 ICMP 에코 요청(ping)을 보냄 (DDOS)
* **Ping of Death**: 혀용된 최대 크기를 초과하는 패킷을 보내 오버플로우 발생시키는 공격 방법

## 정보 보안 침해 공격 관련 용어
* **웜(Worm)**: 스스로를 복제하고 다른 컴퓨터로 자동으로 퍼져나가는 독립적인 악성 소프트웨어
* **랜섬웨어(Ransomware)**: 사용자의 파일을 암호화하여 접근할 수 없도록 만든 후, 이를 해제하기 위해 금전을 요구하는 악성 소프트웨어
* **백도어(Backdoor)**: 시스템이나 소프트웨어에 몰래 설치된 비정상적인 접근 경로 (백도어 탐지 도구: Tripwire)
* **세션 하이재킹(Session Hijacking)**: 세션을 가로채러 불법적으로 **사용자의 권한을 탈취**하는 공격 기법이다.

## Network
### Internet (Inter Connected Network)
> TCP/IP라는 통신 규약을 기반으로, 전 세계의 수많은 컴퓨터와 네트워크들을 하나로 연결한 거대한 컴퓨터 통신망 (논리적/ 물리적 인프라 그 자체)

### Web (World Wide Web)
> Internet 이란 통신망 위에서 동작하는 Service 중 하나로, Text, Image, Stream Data 등 멀티미디어 정보를 Hypertext 로 연결하여 정보를 공유할 수 있는 service

### Web 3.0
> Blockchain 등 분산화 기술을 기반으로 하여, 거대 플랫폼 기업이 아닌 개인이 데이터를 소유하고, AI 등 데이터의 의미를 이해하여 맞춤형 정보를 제공하는 탈중앙화된 지능형 웹

어떻게보면 Google 을 거부하기 위한 도전이라고도 할까나

### Network
> 두대 이상의 컴퓨터나 기기들이 데이터를 주고받을 수 있도록 통신망으로 연결된 구조

### bps (bits per second)
> 1초 동안 전송할 수 있는 bit 수
1. **Kbps**: 1초당 $10^3$ 비트를 전송하는 속도
2. **Mbps**: 1초동안 $10^6$ 비트를 전송할 수 있는 속도, 3G 통신이나 일반적인 유선 인터넷 기준
3. **Gbps**: 1초동안 $10^9$ 비트를 전송하는 속도, 초고속 5G 통신망, 기업용 초고속 LAN 망에서 사용

### LAN (Local Area Network)
> 학교, 회사, 집, PC 방 등 비교적 좁은 지역 내에서 컴퓨터와 기기들을 연결한 통신망
* **관리 주체**: 주로 개인이나 단일 기관(회사, 학교)이 직접 소유하고 관리함 (집에서 직접 공유기를 설치하는것도 포함)
* **성능**: 거리가 짧아 전송 속도가 매우 빠름 (100Mbps ~ 10Gbps)

### MAN (Metropolitan Area Network)
> LAN 보다는 크고 WAN 보다는 작은, 하나의 City 규모를 연결
* **관리 주체**: 여러 개의 LAN을 연결하기 위해 지역 통신 사업자나 컨소시엄이 구축하여 서비스하는 경우가 많음
* **성능**: LAN과 유사하거나 약간 낮은 수준의 고속 통신을 지원

### WAN (광대역 통신망, Wide Area Network)
> 국가, 대륙 등 넓은 지리적 영역을 연결하는 거대한 통신망
* **관리 주체**: 개인이 구축할 수 없으며, KT, SKT와 같은 ISP(인터넷 서비스 제공자, 통신사)가 거대한 망을 구축하고 사용자는 요금을 내고 이를 빌려 씀
* **성능**: 신호가 멀리 가야 하므로 LAN에 비해 상대적으로 전송 속도가 느리고, 통신 오류나 데이터 손실(지연) 발생률이 높음

### Gateway
> 서로 다른 Protocol 이나 Network 환경을 사용하는 통신망들을 이어줄 때, 데이터를 변환하여 통과시켜 주는 출입구 역할을 하는 Network 장비

### 유선 인터넷
> 광케이블, 구리선 등 물리적인 선을 통해 데이터를 주고받는 Network, 안정성과 속도가 뛰어남

### 무선 인터넷
> 전파나 적외선 등을 이용하여 선 없이 데이터를 주고받는 Network 기술, (WIFI, Bluetooth)

### 이동 통신 (Cellular Network)
> 사용자가 자동차를 타거나 걸어서 이동하는 중에도 기지국을 넘나들며 끊김 없이 무선 통신을 할 수 있게 해주는  Service (LTE, 5G)

### 위성 인터넷
> 지상에 설치된 기지국 대신, 지구 궤도를 도는 인공위성을 매개로 인터넷 서비스를 제공하는 기술, 기지국 설치가 어려운 산간 오지나 바다에서도 Internet 서비스 이용 가능

### IoT (사물인터넷, Internet of Things)
> 가전제품, 자동차, 센서 등 세상의 다양한 사물들에 통신 기능을 내장하여, 인터넷을 통해 사물과 사물, 사람과 사물이 데이터를 주고받으며 상호작용 하는 기술

### 초연결 (Hyper-Connectivity)
> 스마트폰, 사물인터넷, 통신망 등을 통해 사람, 기기, 데이터 등 사회의 모든 요소가 거미출처럼 촘촘하게 상시 연결되어 있는 상태나 사회를 의미

### Protocol
> 컴퓨터나 네트워크 기기들이 서로 데이터를 주고받기 위해 미리 정해놓은 공통된 규칙이나 약속

### OSI 7 Layer
> 국제표준화기구(ISO)에서 컴퓨터 네트워크 통신 과정을 7개의 단계로 나누어 정의한 네트워크 표준 모델

## 멀티미디어
> Multi 와 Media 의 합성어로, 문자, 음성, 음악, 영상 등의 다양한 정보를 컴퓨터를 통해 통합하여 활용할 수 있도록 해주는 기술

멀티미디어의 4가지 성립 조건
1. 두 가지 이상의 미디어를 동시에 사용해야 함
2. 이를 처리하기 위한 하나의 시스템을 사용해야 함
3. 사용자와 시스템이 서로 소통할 수 있는 쌍방향성을 가져야 함
4. 시스템을 통해 유의미한 정보를얻을 수 있어야 함

### MPEG
> Moving Picture Experts Group, 동영상 표준 코드 방식의 제정을 위해 출범

1. **공간적 중복성 제거**: 한 장의 사진 안에 인접한 픽셀들은 서로 비슷한 생상과 명암을 가질 확률이 높다는 점을 이용
2. **시간적 중복성 제거**: 동영상에서 바로 이어지는 앞 프레임은 배경 등 변하지 않는 부분이 대부분이라는 점을 이용
3. **통계정 중복성 제거**: 데이터에서 자주 나타나는 패턴은 짧은 코드로, 가끔 나타나는 패턴은 긴 코드로 변환하여 전체 용량을 줄이는 방식




## 최신 컴퓨터 응용 기술
### 빅데이터
> 기존의 DBMS 로는 처리하기에 너무 방대한 데이터 집합을 의미

**4가지 특성**
1. **Volume**: 사이즈
2. **Velocity**: 빠른 생성 속도
3. **Variety**: 다양한 포맷
4. **Veracity**: 부정확하거나, 불완전한 데이터가 포함될 수 있음

### 메타버스
> 가상/초월을 뜻하는 메타(Meta)와 우주를 뜻하는 유니버스(Universe)의 합성어로, 아바타를 통해 실제 현실과 같은 사회, 경제, 교육, 문화 활동을 할 수 있는 3차원 가상 공간 플랫폼

### AR, VR, MR
1. **AR(증강현실)**: 실제 현실을 배경으로 삼고 그 위에 이미지나 문자를 겹쳐서 보여주는 기술 (포켓몬 고)
2. **VR(가상현실)**: 실제 현실은 전혀 없이, 컴퓨터가 만들어낸 100% 가상의 환경 속으로 사용자가 직접 들어간거처럼 느끼게 해주는 기술
3. **MR(혼합현실)**: 현실 세계와 가상 세계가 단순히 겹치는 것을 넘어, 상호작용하며 완전히 혼합된 상태
4. **XR(확장현실)**: 위의 모두를 통틀어 부르는 포괄적인 개념

### Wearable Device
> 착용할수 있는 기기로 신체 특정부위에 간편하게 착용하는 시계나 밴드 등을 포함

### Ubiquitous
> 라틴어로 '어디든지 존재한다'는 뜻으로, 사용자가 언제 어디서나 네트워크에 접속된 컴퓨터를 자신의 컴퓨터처럼 사용할 수 있는 환경을 말함, Network 접속이 필수적이며 컴퓨터를 쓴다는 인식조차 못 할 만큼 자연스럽게 일상에 스며드는 특징을 가짐


### IT 융합 컨버전스
> 정보통신기술(IT)이 기존의 다른 전통 산업(자동차, 의료, 건설, 농업 등)과 결합하여 새로운 가치와 서비스를 창출하는 것

* `IT + 방송/통신` = IPTV, 스마트 TV
* `IT + 교통` = ITS(지능형 교통 시스템), BIS(버스 정보 시스템)


## AI
> 인간의 지적 능력(학습, 추론, 지각, 자연어 이해 등)을 컴퓨터를 통해 소프트웨어로 구현하여 모방하는 기술

* **학습**: 대량의 데이터를 분석해 모델이 스스로 규칙을 찾아내는 과정
* **추론**: 학습된 모델이 새로운 데이터와 마주했을 때 정답을 예측하거나 결론을 내리는 과정

### Physical AI
> 소프트웨어를 넘거, 로봇, 자율주행차, 드론 등 물리적인 신체(Hardware)를 가지고 현실 세계의 환경을 직접 인지하고 물리적으로 상호작용하며 움직이는 인공지능

### Rule-based
> 인간 전문가가 미리 입력해 둔 수많은 Rule(if-else)과 논리를 바탕으로 컴퓨터가 상황을 판단하고 결론을 도출하는 방식

1. 전문사 시스템 (의사, 자동차 진단)
2. 게임 AI
3. 자연어 처리 (기계 번역)

### 신경망 (Neural Network)
> 인간 뇌의 신경세포(뉴런)들이 연결된 구조를 수학적으로 모방한 모델, 방대한 양의 데이터를 주고 컴퓨터가 스스로 패턴과 규칙을 찾아내어 학슴(Learning) 하도록 함

### Deep Learning Framework
* **PyTorch**: Meta가 개발한 오픈소스 프레임워크. 파이썬(Python)과 문법이 매우 유사하고 코드가 직관적이어서 최근 학계, 연구자, 학생들 사이에서 대세로 자리 잡고 있음
* **TensorFlow**: Google이 개발한 오픈소스 딥러닝 프레임워크. 최근엔 TensorFlow 를 그대로 사용하기보단 더 쉽게 사용할 수 있도록 wrapping 한 Keras 를 도입하고 있음

### 시스템의 4가지 요소
1. **Model**: 신경망, 다층 퍼셉트론, 전문가 시스템 등
2. **Hardware and Software**: GPU, SPARK, PyTorch 등
3. **학습과 실행용 프로그래밍 언어**: Python
4. **응용 분야**: LLM, 자연어 처리, 음성 인식 등




## 전자상거래 (E-Commerce)
> 개인, 기업, 정부 등 경제 주체 간에 컴퓨터와 통신망(인터넷)을 이용하여 상품과 서비스를 교환하는 방식

**5가지 분류**
1. **B2B**: 기업과 기업
2. **B2C**: 기업과 소비자
3. **B2G**: 기업과 정부
4. **C2G**: 소비자와 정부 (민원처리, 세금납부)
5. **C2C**: 소비자와 소비자 (중고거래)

**전자상거래 관련 핵심 관리 기술**
1. EDI: 상거래에 필요한 문서를 합의된 표준을 사용하여 통신망으로 교환하는 기술
2. ERP: 인사, 생산, 자재, 영업 등 기업 내 모든 부서의 정보를 통합 관리하는 시스템
3. SCM: 제조, 물류, 유통업체의 상품 흐름을 한눈에 파악하여 재고를 최적화하는 시스템

**주요 응용 분야**
1. VOD
2. 인터넷 뱅킹

## 프로그래밍 언어
> 주어진 문제를 해결하기 위한 인간과 컴퓨터 사이에서 의사소통을 가능하게 해주는 인공적인 언어
* **간결성**
* **직교성**: 각 구성 요소가 상호 독립적이라는 의미
* **가독성**
* **적확성**
* **기계 독립성**: 서로 다른 컴퓨터 상에서 똑같은 결과를 도출해야함

**저급언어**
* 기계어
* 어셈블리 언어

**고급언어**
* C, Cpp
* Python
* Java
* ...

### Compiler
> **고급 언어 프로그램(Source)** -> `Compiler` -> **기계어 프로그램 (Object, 목적 프로그램)** -> `Loader` -> **메모리에 기계어 프로그램 로드** -> `실행` -> 결과

* 프로그램 전체를 한꺼번에 번역
* 번역 후 빠른 시간 내에 전체를 실행 가능
* 프로그램 일부를 수정하는 경우에도 다시 전체를 컴파일 해야 함

> c, cpp, rust

### Interpreter
> 작성된 프로그래밍 소스 코드를 미리 기계어로 번역하지 않고, 실행할 때 한 줄씩 직접 해석하여 즉시 실행하는 프로그램 또는 번역기

* 실행될 행 단위마다 번역
* 큰 Memory 가 필요하지 않으며 번역 과정이 비교적 간단
* 실시간으로 번역해야 하는 오버헤드가 존재

> python, JavaScript, Ruby

## Security
### Blockchain
> 중앙 관리자(은행 등) 없이, 거래 참여자 모두가 데이터를 블록(Block) 단위로 묶어 체인처럼 연결하여 분산 보관하는 P2P 분산 장부 기술, 해킹이나 데이터 위변조가 사실상 불가능
* Bitcoin
* NFT

### 대칭키 vs 비대칭키

* **대칭키(비밀키)**: 암호화 키와 해독 키가 같음 (속도 빠름, 키 전달 위험)
  * DES
* **비대칭키(공개키)**: 암호화 키와 해독 키가 다름 (속도 느림, 매우 안전)
  * RSA

**비대칭키 정보 전달 과정**
1. 수신자 B 는 자신만의 공개키와 개인키 한쌍을 만듬, 공개키를 Network 상에 공개
2. 송신자 A 는 B 의 공개키를 가져와 암호화
3. A 는 암호화된 데이터를 네트워크를 통해 B 에게 전송
4. B 는 전달받은 암호문을 자신만 가지고 있는 개인키를 사용하여 복호화

### 주요 사이버 보안 위협
* **스미싱 (Smishing)**: SMS(문자메시지)와 피싱(Phishing)의 합성어. 택배 배송 등의 문자에 포함된 링크를 누르면 악성코드가 설치되는 수법
* **피싱 (Phishing)**: 금융/공공기관을 사칭해 가짜 사이트로 유도하여 개인정보를 빼내는 수법
* **파밍 (Pharming)**: 정상 주소를 입력해도 가짜 사이트로 강제 접속되게 만드는 피싱의 진화형
* **랜섬웨어 (Ransomware)**: 시스템을 감염시켜 파일을 암호화해 열지 못하게 한 뒤, 돈(몸값)을 요구하는 악성 프로그램
* **DDoS (디도스)**: 수많은 좀비 PC를 이용해 한 서버에 엄청난 트래픽을 집중시켜 서버를 마비시키는 공격

## Database
> 서로 관련 있는 데이터들을 효율적으로 관리하기 위해 수집된 데이터들의 집합체

> DBMS: DB를 보다 편리하게 정의, 생성, 조작할 수 있도록 해주는 범용 소프트웨어 시스템

**4가지 특징**
1. **실시간 접근성**: 주어진 질의를 실시간에 처리할 수 있어야 함
2. **계속적 변화**: 새로운 데이터 삽입, 기존 데이터 삭제, 갱신 등 현재의 정확한 데이터를 항상 유지해야함
3. **동시 공유**: 여러 목적을 가진 사용자가 동시에 다양한 방법으로 접근하여 이용할 수 있어야 함
4. **내용에 의한 참조**: 데이터의 실제 물리적 위치가 아닌 Data 의 값(이름) 으로 찾을 수 있어야 함

**DBMS 의 특징 및 역할**
1. 데이터 중복 최소화
2. 데이터 무결성 유지
3. 보안 및 권한 제어
4. 장애 복구

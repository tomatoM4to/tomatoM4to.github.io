# 임베디드 시스템의 특징
하드웨어의 의존성과 제품 양산 측면에서의 제한 사항이 존재한다.

* 최적화

* 안정정성
    * 내구성이 좋은 부품, 하지만 최적화가 잘 돼어있는 부품과의 트레이드 오프가 있음
    * 요즘은 네트워크에 다 연결이 돼어 있어 업데이트가 가능 하지만, 보통 외외부의 제어ㅡㄹ수행하기 어려운 환경에서 시스템 에러를 발생시키지 않는 안정성이 필요, 이또한 성능과의 트레이드 오프가 발생

* 실시간 처리

* 하드웨어 제어 지식 요구
    * 예를들어 어떤 리모모콘의 버튼을 눌렀을때, 레지스터에 어떤 변화가 있는지지에 대한 정보


해당 수업에선 라즈베리 파이로 수업하지만, 라즈베리파이는 임베디드 중에서 성능이 좋좋은편이라

# 임베디드 시스템 구조
ppt 그림은 표준은 아아님

과거에는 인터넷을 배제했지만 지금은 인터넷을 고려함

요즘은 AI가 있지만 사실 그 그림자 속에서 iot도 계속 성장중!

해당 그림은 냉장고를 타겟으로 잡음

온도를 잡고싶을때 어떤식으로 작동하면, 중간이 센서임

커뮤니티 채널은 통신 방식에 해당함, 냉장고 같은 경우 다 유선으로 돼있음

TCPIP 기반으로 수집을 할수도 있음

Main Pocessing uit: CPU애 해당

UI를 통해 온도 제어 같은 개입 가능

Actuator: 모터, LED 같은출력장치를 의미함

시스템 외적으론 인터넷을 사용해, 모바일로 확인 하거나 클라우드에 업ㄹ드 해 제품 개선으로 이어지지기도 함

임베디드나 IOT 같은 경우 가장 많이 나오는 문제는 사생활 보호, 실시간 처리 가 있음

모든걸 다 활용용하지는 않지만 이 그림림을 이해하는게 목표라 할수 있음

# 파일 시스템
사용자들이 사용하는 파일들을 관리하는 운영체제의 한 부분을 의미한다.

어렵게 생각할 필요 없다.

#### 파일 시스템의 구성
1. **파일**: 연관된 정보의 집합
2. **디렉토리 구조**: 시스템 내 파일들의 정보를 구성 및 제공하는것, 쉽게말해 폴더를 의미한다.
3. **파티션**: 디렉토리들의 집합을 논리적/물리적으로 구분하는것, 쉽게말해 C드라이브, D드라이브 같은것을 의미한다.

***

# File 이란?
전공자로서 엄밀하게 파일을 정의해보면 보조 기억 장치에 저장된 연관된 정보의 집합, 보조기억 장치에 할당하는 최소 단위 로 정의해볼수 있다.

좀더 물리적으로 정의하면, 바이트 들의 집합(Sequence of bytes) 라 정의해볼수 있다.

#### 내용에 따른 분류
실행파일(Program file) 과 데이터를 담는 Data file 로 구분해볼수 있다.


#### 형태에 따른 분류
Text file과 Binary file로 구분해볼수 있다, Text file는 눌렀을때 사람이 읽을수 있는 형태의 파일이고 Binary file은 사람이 읽을수 없는 0과 1로 이루어진 파일이다.



#### 파일의 속성
우클릭을해 속성을 클릭하면 파일이 가지고 있는 여러가지 속성을 볼수 있다.

파일의 속성은 여러가지가 있으며, 대표적으로 파일의 이름, 크기, 생성일, 수정일, 소유자, 권한, 파일의 위치 등이 있다.


#### File operations
파일을 Create, Write, Read, Reposition, Delete 등의 연산을 파일 연산이라고 한다.

중요한건 이러한 다양한 연산들을 OS가 제공해야 한다는 거다. 즉 File operation들에 대한 system call을 제공하는것은 OS의 주요 역할 중 하나이다.

# 파일 접근 방법[File Access Methods]
크게 3가지로 나눌수 있다.

1. **순자 접근[Sequential access]**: File을 record(or bytes) 단위로 순서대로 접근 하는 방법이다.
2. **직접 접근[Direct access]**: 원하는 Block을 직접 찾아가 접근하는 방법이다.
3. **색인 접근[Index access]**: Index를 참조하여, 원하는 block을 찾은 후 데이터에 접근하는 방법이다.

# 파일 시스템 조직[File System Organization]
#### 디렉토리[Eirectory]
파일들을 분류, 보관하기 위한 개념이다. 쉽게말해 폴더라고 생각하면 된다.

디렉토리 또한 OS가 System call을 통해 여러가지 연산을 제공해야 한다.

대표적인 연산으론 다음과 같다.

1. **Search for a file**: 파일을 찾는 연산
2. **Create a file**: 파일을 생성하는 연산
3. **Delete a file**: 파일을 삭제하는 연산
4. **List a directory**: 디렉토리의 파일 목록을 보여주는 연산
5. **Rename a file**: 파일의 이름을 변경하는 연산
6. **Traverse the file system**: 파일 시스템을 탐색하는 연산

#### 파티션[Partition]
파일 시스템을 논리적/물리적으로 구분하는것을 의미한다.

예를들어 지금 자신의 컴퓨터에는 디스크가 몇개가 있나?

하나를 쓰는경우, 두개를 쓰는경우, 세개를 쓰는경우가 있다.

하나를 쓰는경우에도 C드라이브, D드라이브 나누어 쓰는 경우가 있다.

혹은 두개의 물리적인 디스크를 하나의 C드라이브로 사용하는 경우도 있다.

이처럼 디스크 를 논리적으로 나누어놓은 디스크, 즉 물리적인 디스카가 아닌 **논리적인 디스크**를 파티션 또는 Virtual disk, minidisks, volumes 라고 한다.

#### 마운팅[Mounting]
마운팅이라는 용어는 리눅스(안드로이드)에서 많이 사용되는 용어이다.

마운팅 이라는건 현재 File system 에 다른 File system을 연결하는것을 의미한다.

자주 볼수있는 예를 들자면, 안드로이드 폰에 SD카드를 꽂거나 빼면, SD 카드가 마운트 돼었습니다, 마운트 해제돼었습니다. 라는 메세지가 나오기도 한다.



실제 안드로이드 핸드폰에서서 파일탐샘기를 통해 /mnt 디렉토리를 볼수 있을거다. 우리가 SD 카드에 접근을 할때 보통 /mnt/[sdcard] 를 통해 접근을 하게 된다.

이는 폰이 가진 기본적인 파일 시스템에서 추가적으로 SD 카드에 접근하고 싶은데, 이를 접근하기 위해 파일시스템 내의 어떤 디렉토리 에 SD카드를 연결 해주는 거다.

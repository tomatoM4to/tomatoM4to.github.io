# SQL 소개
**SQL(Structured Query Language)**은 데이터베이스에서 <R>데이터를 관리하고 조작하기 위한 특수 목적의 프로그래밍 언어</R>입니다. 스키마 생성과 수정, 데이터 검색, 삽입, 수정, 삭제 등 다양한 데이터베이스 작업을 수행할 수 있습니다.

## SQL의 특징
1. **특수 목적 언어:** SQL은 데이터베이스에서 데이터를 추출하고 관리하는 데 특화되어 있습니다. 일반적인 프로그래밍 언어가 다양한 문제를 해결하는 데 사용되는 것과는 달리, SQL은 데이터베이스 작업에 중점을 둡니다.
2. **표준화된 언어:** 대부분의 관계형 데이터베이스 관리 시스템(RDBMS)에서 사용됩니다. `Oracle`, `MySQL`, `PostgreSQL`, `Microsoft SQL Server` 등 다양한 DBMS에서 SQL을 지원합니다.
3. **DBMS 간 호환성:** DBMS에 따라 약간의 차이가 있을 수 있지만, 대부분 비슷한 SQL 문법을 사용합니다. 이로 인해 한 DBMS에서 학습한 SQL 지식을 다른 DBMS에서도 쉽게 적용할 수 있습니다.
4. **낮은 학습 곡선:** 다른 프로그래밍 언어와 달리, SQL은 비교적 배우기 쉽고 직관적입니다. 기본 문법을 익히면 다양한 데이터베이스 작업을 수행할 수 있습니다.

## SQL 기능에 따른 분류
SQL은 데이터베이스 작업을 수행하는 데 사용되는 다양한 기능을 제공합니다. 주요 기능에 따라 SQL 문을 다음과 같이 분류할 수 있습니다. 프로그래밍 언어로 예로 들면 반복문, 조건문 등과 같이 기능을 나누는것과 비슷합니다.

1. **DDL(Data Definition Language):** 데이터 정의어
    * **용도:** 데이터베이스 구조를 정의하고 수정하는 데 사용합니다.
    * **주요 작업:** 테이블의 생성, 수정, 삭제 및 관계의 구조 생성
    * **사용 시기:** 데이터베이스를 처음 설계하거나 구조를 변경할 때 사용합니다.
    * **주요 명령어:** `CREATE`, `ALTER`, `DROP`
    * **중요 포인트:** 데이터베이스 설계 시 신중하게 계획하여 추후 수정과 삭제를 최소화하는 것이 중요합니다.

2. **DML(Data Manipulation Language):** 데이터 조작어
    * 용도: 테이블의 데이터를 조작하는 데 사용합니다.
    * 주요 작업: 데이터 검색, 삽입, 수정, 삭제
    * 특징: SQL에서 가장 빈번하게 사용되는 명령어들입니다.
    * 주요 명령어: `SELECT`, `INSERT`, `UPDATE`, `DELETE`
    * 참고: `SELECT` 문은 특별히 질의어(query)라고 불립니다.

3. **DCL(Data Control Language):** 데이터 제어어
    * 용도: 데이터베이스의 접근 권한을 관리합니다.
    * 주요 작업: 사용자에게 권한을 부여하거나 회수
    * 주요 명령어: `GRANT`, `REVOKE`

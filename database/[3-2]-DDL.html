테이블 생성과 데이터 추가

이 장에서는 테이블을 만들고 데이터를 추가하는 방법을 알아보겠습니다. 데이터베이스를 설계할 때는 테이블을 먼저 만들고, 데이터를 추가하는 순서로 진행합니다. 이를테면 집을 짓는것과 비슷합니다. 먼저 설계를 하고, 테이블을 만들고, 데이터를 추가하는 순서입니다.

CREATE TABLE

CREATE TABLE은 테이블을 생성하는 명령어입니다. 테이블을 생성할 때는 테이블 이름, 각 속성의 이름과 데이터 타입을 지정해야 합니다. 중요 포인트는 PRIMARY KEY 및 FOREIGN KEY또한 CREATE TABLE문에서 지정해야 한다는 것입니다. 기본 구조는 다음과 같습니다.

CREATE TABLE [테이블 이름] (
    [속성1 이름] [데이터 타입] [제약조건],
    [속성2 이름] [데이터 타입] [제약조건],
    PRIMARY KEY ([기본키 속성 이름]),
    FOREIGN KEY ([외래키 속성 이름]) REFERENCES [참조할 테이블 이름]([참조할 속성 이름])
)

데이터 타입

데이터 타입은 속성이 가질 수 있는 값의 종류를 정의합니다. 데이터 타입을 정의할때 주의해야할 부분이 있습니다. 데이터 타입은 DBMS마다 약간의 차이가 있을 수 있으므로 사용하는 DBMS의 문서를 참고해야 합니다. 대표적인 데이터 타입은 다음과 같습니다.

SQL설명
INT32bit 정수 값을 나타내는 데이터 타입입니다.
NUMERIC매우 크고 정밀한 수를 저장할수 있는 타입입니다. 소수점 왼쪽에 최대 131,072개의 숫자, 소수좀 오른쪽에 최대 16,383개의 숫자를 저장할 수 있습니다.
CHAR(n)고정길이 문자열을 나타내는 데이터 타입입니다. 문자열의 최대 길이는 n입니다.
VARCHAR(n)최대 길이가 n인 가변길이 문자열을 나타내는 데이터 타입입니다. 문자열을 저장하는 가장 흔한 방식입니다.
DATE연도, 월, 일을 나타내는 데이터 타입입니다.
TIME시간(시, 분, 초)을 나타내는 데이터 타입입니다.

Primary Key, Foreign Key

백문불여일견 이라고 말로 설명하는것보단 코드를 바로 보여드리겠습니다. SQL의 주요 특징으론 대소문자를 구분하지 않는다는 것입니다. 하지만 보통 키워드는 대문자로 쓰고 나머지는 소문자로 쓰는 것이 관례입니다.

CREATE TABLE Order (
    order_id INT PRIMARY KEY,
    customer_id INT,
    order_date DATE,
    total_price NUMERIC(10, 2),
    FOREIGN KEY (customer_id) REFERENCES Customer(customer_id)
)

위의 SQL 코드를 보면 Order 테이블을 생성하고 있습니다. order_id 속성은 INT 데이터 타입이며 PRIMARY KEY로 지정되어 있습니다. customer_id 속성은 INT 데이터 타입이며 FOREIGN KEY로 지정되어 있습니다. FOREIGN KEY는 Customer 테이블의 customer_id 속성을 참조하고 있습니다. Customer 테이블은 이미 생성되어 있다고 가정합니다. 이로서 어떤 고객이, 언제, 얼마를 주문했는지를 저장하는 Order 테이블을 생성한 것입니다.

NUMERIC에 대한 추가적인 설명을 붙이자면 정수 부분: 8자리, 소수 부분: 2자리를 저장할 수 있습니다. 즉, 최대 99,999,999.99까지 저장할 수 있습니다.

만약 복합키를 만들고 싶다면 다음과 같이 작성하면 됩니다. 변경점은 PRIMARY KEY에 속성을 여러개 쓰는 것입니다. PRIMARY KRY(order_id) 로 작성하게되면 위의 코드와 동일한 의미가 됩니다.

CREATE TABLE Order (
    order_id INT,
    customer_id INT,
    order_date DATE,
    total_price NUMERIC(10, 2),
    PRIMARY KEY (order_id, customer_id),
    FOREIGN KEY (customer_id) REFERENCES Customer(customer_id)
)

제약조건

제약조건은 데이터베이스의 무결성을 유지하기 위해 테이블에 저장될 데이터를 제한하는 규칙입니다. 이를 통해 잘못된 데이터가 데이터베이스에 입력되는 것을 방지할 수 있습니다. 예를 들어, Order 테이블에서 총 주문 금액total_price이 음수가 되는 것은 논리적으로 맞지 않습니다. 이러한 상황을 방지하기 위해 제약조건을 설정해야 합니다. 다음은 total_price에 제약조건을 추가한 Order 테이블 생성 예시입니다:

CREATE TABLE Order (
    order_id INT,
    customer_id INT,
    order_date DATE,
    total_price NUMERIC(10, 2) CHECK(total_price >= 0),
    PRIMARY KEY (order_id, customer_id),
    FOREIGN KEY (customer_id) REFERENCES Customer(customer_id)
)

이 예시에서 CHECK(total_price >= 0) 제약조건은 total_price가 음수가 되는 것을 방지합니다. 이외에도 다양한 제약조건을 추가할 수 있습니다. 이러한 제약조건은 하나의 속성에 여러개를 추가할 수 있습니다.

CREATE TABLE Order (
    order_id INT,
    customer_id INT,
    order_date DATE,
    total_price NUMERIC(10, 2) CHECK(total_price >= 0) DEFAULT 0 NOT NULL,
    PRIMARY KEY (order_id, customer_id),
    FOREIGN KEY (customer_id) REFERENCES Customer(customer_id)
)

이 예시에서 total_price 컬럼에는 다음과 같은 제약조건이 설정되었습니다.

  1. CHECK(total_price >= 0): 음수 값을 허용하지 않습니다.
  2. DEFAULT 0: 값이 지정되지 않았을 때 기본값을 0으로 설정합니다.
  3. NOT NULL: NULL 값을 허용하지 않습니다.

INSERT INTO

INSERT문은 테이블에 새로운 튜플을 삽입하는 명령입니다. INSERT문은 다음과 같은 구조를 가집니다.

INSERT INTO [테이블 이름]([속성1 이름], [속성2 이름], ...) VALUES([속성1 값], [속성2 값], ...)

위 구조를 보면 INSERT INTO문을 사용할 때는 테이블 이름과 속성 이름, 그리고 속성 값이 필요합니다. 다음은 Order 테이블에 데이터를 추가하는 예시입니다.

INSERT INTO Order(order_id, customer_id, order_date, total_price) VALUES(1, 1, '2021-01-01', 100.00);
INSERT INTO Order(order_id, customer_id, order_date, total_price) VALUES(2, 2, '2021-01-02', 200.00);
INSERT INTO Order(order_id, customer_id, order_date, total_price) VALUES(3, 3, '2021-01-03', 300.00);

INSERT INTO Order VALUES(4, 4, '2021-01-04', 400.00);
INSERT INTO Order(customer_id, order_date) VALUES(5, '2021-01-05');
INSERT INTO Order(customer_id, order_date) VALUES(5, '2021-01-05', -100.00);

위의 코드를 보면 Order 테이블에 데이터를 추가하는 예시입니다. INSERT INTO문을 사용할 때는 속성 이름을 지정하여 값을 추가할 수 있습니다. 속성 이름을 지정하지 않으면 테이블의 속성 순서대로 값을 추가합니다. 이때 주의할 점은 INSERT INTO문을 사용할 때는 속성의 순서와 개수가 일치해야 합니다. 만약 일치하지 않으면 오류가 발생합니다. 에러 메시지를 보면 어떤 부분이 잘못되었는지 알 수 있습니다.

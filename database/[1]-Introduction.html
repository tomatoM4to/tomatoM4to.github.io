소개

이 책은 실습 중심의 책으로 개발자가 데이터베이스의 기본적인 개념을 이해하고, 데이터베이스를 설계하고, 데이터베이스를 관리하는 방법을 학습할 수 있도록 구성되어 있습니다. 실습 중심의 책이라는 것은 이론적인 내용을 최소화하고, 실습을 통해 데이터베이스에 대한 이해를 높이는 것을 목표로 합니다. 실습으로 진행할 데이터베이스는 PostgreSQL과 SQLite를 사용합니다.

하지만 이 책으로부터 데이터베이스의 모든 것을 배울 수는 없습니다. 데이터베이스는 매우 방대한 분야이며, 이 책은 데이터베이스의 기본적인 개념과 실습을 통해 데이터베이스에 대한 이해를 높이는 것을 목표로 합니다.

데이터베이스란 무엇인가?

데이터베이스를 공부하기 전 먼저 데이터에 대한 정의부터 시작하겠습니다.

  • 데이터: 관찰의 결과로 나타난 정량적 혹은 정석정인 실제 값
  • 정보: 데이터에 의미를 부여한 것
  • 지식: 사물이나 현상에 대한 이해

위 내용을 토대로 데이터베이스를 정의하면 정보, 지식을 추출하기 위한 데이터의 집합이라고 할 수 있습니다. 사실 데이터나 정보를 정의하는 것은 추상적인 개념이기도 하고 데이터베이스는 엄밀하게 무엇이 데이터베이스다! 라고 정의되지 않았습니다. 보통 데이터베이스를 공부한다는 것은 값을 저장, 탐색, 수정을 안전하고 효율적이게 하기 위한 software를 공부한다, 라는 것이며 이 책 또한 이를 소개하는 책이라는 것만 이해해도 충분합니다.

역할

데이터베이스의 역할은 크게 2가지로 나눌수 있습니다.

  1. Read: 검색
  2. Write: 변경

데이터베이스는 결국 수억, 수십억의 데이터를 빠르고 안전하게 저장하고, 검색하고 변경하는 것이 목적입니다.

데이터베이스의 특징

데이터베이스에 데이터를 저장할 때는 중복을 최소화하며, 특정한 목적과 함께 여러 사람이 공유할 수 있도록 저장해야 합니다.

자세한 특징은 다음과 같습니다. 하지만 이러한 특징들 중, 2번과 3번 특징은 별로 의미 있는 특징은 아니라 할 수 있겠습니다, 너무나 당연하게 컴퓨터에 저장하고, 쓸모없는 데이터를 저장할 일은 별로 없기 때문이죠.

  1. 통합된 데이터(Intergrated Data)
    • 데이터를 통합하는 개념으로, 데이터 중복을 최소화하고 데이터의 일관성을 유지하는 것을 말합니다.
  2. 저장된 데이터(Stored Data)
    • 컴퓨터가 접근할 수 있는 저장 매체에 저장된 데이터를 말합니다.
  3. 운영 데이터(Operational Data)
    • 조직의 업무를 수행하는데 있어서 반드시 필요한 데이터를 말합니다.
  4. 공용 데이터(Shared Data)
    • 한 사람 또는 한 업무를 위해 사용되는 데이터가 아니라 공동으로 사용되는 데이터를 의미합니다.

데이터베이스에 저장될 데이터의 특징을 살펴봤으니 이제 데이터베이스의 특징에 대해 살펴보겠습니다. 데이터베이스의 몇 가지 특징은 다음과 같습니다. 1, 2, 3번은 앞서 계속 언급했던 내용이고, 4번만 새롭게 알아볼만한 내용입니다.

  1. 실시간 접근성(Real-time Access)
    • 데이터베이스는 사용자의 요구에 수 초 내로 응답할 수 있어야 합니다.
  2. 계속적인 변화(Continuous Evolution)
    • 데이터베이스는 새로운 데이터를 추가하거나 기존의 데이터를 수정하거나 삭제할 수 있어야 합니다.
  3. 동시 공유성(Concurrent Sharing)
    • 여러 사용자가 동시에 데이터베이스에 접근할 수 있어야 합니다.
  4. 내용 참조(Content Reference)
    • 데이터베이스에 저장된 데이터는 데이터의 물리적인 주소가 아닌 데이터 값에 따라 참조되어야 합니다.
SELECT name, age
FROM users

해당 SQL문은 users 테이블에서 name과 age 컬럼을 조회하는 SQL문입니다. 이렇게 데이터베이스에 저장된 데이터를 조회할 때 데이터의 물리적인 주소가 아닌 데이터 값에 따라 참조하는 것을 말합니다. 이 코드에 대해 이해하지 못해도 상관없습니다. 3장부터 SQL문에 대해 자세히 다루기 때문입니다.

데이터베이스 시스템의 구성

보통 데이터베이스 시스템은 일반 사용자가 직접 접근하는 것을 허용하지 않도록 설계합니다. 그리고 데이터베이스 시스템을 설계하는 사람 또한 보통 실제 데이터베이스에 접근하지 않습니다. 이 단락에서는 데이터베이스 관리자(DBA)와 데이터베이스 관리 시스템(DBMS)에 대해 알아보겠습니다.

[1장 그림 필요]

  1. 데이터베이스 관리자(DBA)
    • 데이터베이스 시스템을 설계하고 구축하며, 데이터베이스 시스템의 모든 책임을 집니다.
  2. DBMS
    • 데이터베이스 관리 시스템은 데이터베이스를 관리하는 소프트웨어를 말합니다.

Datebase Management System(DBMS)

DBMS는 데이터베이스를 관리하는 소프트웨어를 말합니다. DBMS는 데이터베이스를 생성하고, 수정하고, 삭제하며, 데이터베이스에 저장된 데이터를 조회하고, 수정하고, 삭제하는 기능을 제공합니다. 보통 데이터베이스를 관리한다는 것은 DBMS를 관리한다는 것을 의미합니다. 앞서 설명한 것처럼, 개발자, 관리자(DBA)들은 물리적인 실제 데이터베이스에 직접 접근하지 않고 DBMS를 통해 데이터베이스를 관리합니다. 데이터베이스 전문가가 된다는 것 또한 사실 DBMS를 잘 다루는 것을 의미합니다. 데이터베이스를 검색하면 가장 많이 나오는 것이 Oracle, MySQL, PostgreSQL 같은 DBMS들이며, 이 책에서는 PostgreSQL과 SQLite를 사용합니다.

또 DBMS는 앞서 설명한 실시간 접근성, 계속적인 변화, 동시 공유성, 내용 참조 같은 기능들을 기본적으로 제공하면서, 데이터베이스의 백업과 복구, 데이터베이스의 보안, 데이터베이스의 성능 튜닝, 데이터베이스의 병행 제어, 데이터베이스의 데이터 무결성 같은 보다 전문적인 기능까지 제공합니다.

다음으로 DBMS의 기능에 대해 간략히 소개하고 넘어가겠습니다.

  1. 데이터 정의: 데이터의 구조를 정의하고 데이터 구조에 대한 삭제 및 변경 기능을 수행합니다.
  2. 데이터 조작: 데이터베이스에 저장된 데이터를 조회하고, 삽입하고, 수정하고, 삭제하는 기능을 수행합니다.
  3. 데이터 추출: 데이터베이스에 저장된 데이터를 외부(응용 프로그램)로 추출하는 기능을 수행합니다.
  4. 데이터 제어: 데이터베이스에 저장된 데이터에 대한 접근 권한, 백업과 회복, 동시성 제어 등의 기능을 지원합니다.

DBMS와 File System

컴퓨터가 대중에게 공급되기 시작한 1950년대부터 1960년대까지는 데이터를 저장하기 위해 파일 시스템을 사용했습니다. 파일 시스템은 우리가 일상적으로 사용하는 방식으로, 데이터를 파일로 저장하고, 파일을 디렉토리로 관리하는 방식입니다. 지금도 작은 데이터는 파일 시스템을 사용하여 저장하고 있지만 만약 데이터가 수억, 수십억 개 이상이 되면서부터 파일 시스템은 데이터를 효율적으로 관리할 수 없게 되었습니다. 이러한 문제를 해결하기 위해 DBMS가 등장하게 되었습니다.

파일 시스템이 나쁘다는 건 아닙니다. 사실 우리가 일상적으로 사용하는 데이터를 DBMS로 저장하는 것은 오히려 비효율적입니다. 심지어 수십만, 수백만 정도의 데이터 또한 파일 시스템으로 데이터를 저장하는 것이 효율 면에선 오히려 DBMS보다 좋을 수 있습니다. DBMS는 CPU와 메모리를 많이 사용하기 때문입니다. DBMS는 인간이 더 이상 컨트롤하기 어려운 수준의 데이터를 관리하기 위한 매우 무겁고 전문적인 software라는 것을 이해하면 됩니다.

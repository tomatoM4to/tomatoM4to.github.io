Network Security 소개

가볍게 Network Security 가 왜 필요한지, 어떤 공격이 있었는지, Network Security가 목표로 하는 것은 무엇인지에 대해 알아보겠습니다.

Network Security는 이름에서 알수 있든 Security의 하위 분야입니다.

Network 보안의 필요성

  • 현재 회사, 가정, 개인은 유무선 Network에 24시간 연결되어 있음, 이는 24시간 사이버 공격에 노출되어 있다는 뜻이겠죠.
  • 다양한 보안 위협: 이메일에 첨부된 악성코드 파일 실행을 통한 감염, 외부에서 회사 네트워크에 접속하려는 시도, 회사 정보가 해킹되어 외부로 유출 등등

과게에는 단순히 Network 보안 시스템들을 구축하여 Network의 접근을 통제 하는 것만으로 충분

Network 보안 시스템이라면 Firewall, Auth Server(id, pw로 접근 통제)

현재는 다양한 경로로 Network에 연결된 수많은 단말기로부터 다양한 보안 위협을 탐지하고 대응해야 함

Network 공격 사례

  • 모리스 웜

  • 2003년 1.25, Slammer 공격: 10분만에 75,000 호스트 감염, 인터넷 전체 속도 50% 감소, 국내 최상위 DNS도 공격당함, 전국 대부분의 인터넷 망이 불통, 전 세계에서 동시에 발생

  • 이로 인해 공공기관, 기업들이 정보보안 강화를 위해 예산을 늘리는 등 역량 강화를 위한 노력을 기울임, 하지만 보안 제품만 도입하고 사후 운영이 제대로 되지 않아 아직까지 많은 위험이 존재

  • 대기업이나 정부기관은 그나마 상황이 나아도, 중소기업 같은 경우 기본적인 보안 시스템조차 없는 경우가 많아, 공격을 받았는지, 공격을 받았다면 어떤 경로로 유입되었는지 조차 파악할 수 없는 실정

사이버 범죄 들의 공통점: Network로 연결된 곳에서 발생, 1990 년대부터 필요성을 눈치 체기 시작

지금 현재도 보안 분야에서 Network Security는 가장 중요한 분야 로 여겨짐

정보 보안 목표

기밀성(Confidentiality)

허락되지 않은 사용자가 정보에 접근하지 못하도록 하는 것, 오직 너하고 나하고만 아는것

어떻게 보면 프라이버시 보호와 관계 있음

Network Security 측면에서 기밀성은 시스템 간 안전한 데이터 전송 과 관련이 있음

기밀성을 해치는 가장 일반적인 공격엔 Sniffing(도청) 공격이 있음, 중간에 편지를 열어보는 것과 같은 것

통신의 암호화가 가장 일반적인 보안 대책

무결성(Integrity)

허락되지 않은 사용자가 정보를 수정할 수 없도록 하는 것, 무결하다 라는건 수정되지 않았다는 뜻

처음 보낸 사람이 보낸 내용이 그대로 전달되었는지 보장 해야 함

클라이언트와 서버 간의 데이터가 변조되지 않고 전송되는가 와 관련이 있음

중간에 유요한 다른 연결을 빼앗는 세션 하이재킹, 두 시스템 간의 데이터를 중간에 변조하는 Man in the Middle 공격이 있음

MITM(Man in the Middle) 은 송신자 A와 수신자 B 사이에 제 3자 C가 끼어들어 A - C - B로 데이터를 전달하는 것 그 반대도 가능

이또한 암호화가 가장 일반적인 보안 대책, 애초에 열지 못하면 수정할수 없음, Public Key Infrastructure(PKI) 를 사용

가용성(Availability)

허락된 사용자가 언제든 원할때 정보에 접근할 수 있도록 하는 것

예를들어 동시 허용 접속자 수가 1000인 서버에 10000, 20000 의 가짜 접속을 시도하여 서버를 다운시키는 것

DOS(Denial of Service) 가 대표적인 공격, 서버가 가짜 Request를 처리하느라 정상적인 Request를 처리하지 못하는 것

AI: 뚜렷한 해결책은 없지만, 서버의 부하를 분산시키는 Load Balancer 를 사용하거나, 서버의 부하를 감지하여 가짜 Request를 차단하는 방법이 있음

서버 인증(Server Authentication)

위 3가지는 Security의 기본 3요소, 서버 인증은 Network Security의 추가적인 요소

인증: 나라는것을 증명 하는것

클라이언트가 올바른 서버로 접속하는가 를 의미

서버 인증으로 생기는 문제는 무척 다양하여 명확하게 정의하기 어려움

일반적으로 DNS spoofing 이 있음

  • snipping: 몰래 듣기
  • spoofing: 속이기

DNS spoofing 은, 예를들어 google.com 을 입력하면 DNS 서버에 요청하여 IP 주소를 받아와야 하는데 가짜 DNS 서버에 요청하여 가짜 IP 주소를 받아오는 것

결과적으로 Server pharming 과 같은 공격이 발생(원본 사이트와 구분하기 어려운 가짜 사이트를 만들어 사용자를 속이는 공격)

SSL(HTTPS)을 통해 서버 인증을 하지만, 경고를 보여주고 사용자에게 확인을 요청하는 경우가 있음, 강제적인 서버 인증은 흔치 않음

클라이언트 인증(Client Authentication)

이또한 Network Security의 추가적인 요소

이번엔 서버 입장에서 올바른 클라이언트가 접속을 시도하는가 를 의미

ID/PW를 통한 인증이 가장 대표적인 인증 방법

스푸핑, 세션 하이재킹, 피싱 등등 다양한 공격이 있음

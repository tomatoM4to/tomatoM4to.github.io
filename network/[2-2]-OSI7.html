Protocol

이전 시간에 네트워크의 기본적인 개념을 알아보았습니다. 결론적으로 네트워크는 참여자들 간의 데이터 통신을 위해 만들어진 시스템이라고 할 수 있습니다. 그렇다면 이러한 통신 기능이 제대로 동작하기 위해서는 참여자들 사이에 어떠한 약속이 필요할 것입니다. 예로들면, 한국인과 미국인이 대화를 할때, 한국어로 대화를 하면 미국인은 이해를 할 수 없을 것입니다. 한국인과 미국인이 대화를 하기 위해선 약속된 공통의 언어를 사용해야 합니다.

이렇듯 네트워크에서도 데이터를 주고 받기 위한 약속된 통신 방법이 필요합니다. 이러한 방법을 Network Protocol이라 하며, 네트워크 통신을 하기 위해서 통신에 참여하는 주체들이 따라야 하는 형식, 절차, 규약을 의미합니다.

그렇다면 어떠한 Protocol이 네트워크를 만들때 필요할까요? 한번 편하게 생각해 봅시다.

  1. 메일을 보낼때, 메일 서버와 클라이언트 사이에 어떠한 데이터를 주고 받을지에 대한 약속이 필요합니다.
  2. 게임을 할때, 게임 서버와 클라이언트 사이에 어떠한 데이터를 주고 받을지에 대한 약속이 필요합니다.
  3. 목적지로 데이터를 전송할때, 어떠한 경로로 데이터를 전송할지에 대한 약속이 필요합니다.
  4. 신뢰할 수 있는 데이터 전송 방법이 필요합니다.
  5. ...

이렇듯 네트워크를 구축하기 위해선 다양한 Protocol이 필요합니다. 이러한 Protocol을 단 하나의 Protocol로 구현하는것이 가능할까요? 답은 No 입니다. Protocol을 단 하나의 Protocol로 구현한다는것은 마치 프로그램을 하나의 함수로 구현하는것과 같습니다. 이러한 방식은 매우 비효율적이며, 유지보수가 어렵습니다. 따라서, 네트워크를 구축할때는 각 계층을 나누어 Protocol을 구현합니다.

OSI 7계층이란, 네트워크 통신 기능을 7개의 계층으로 모델링한 개방형 시스템 상호 연결(Open Systems Interconnection) 참조 모델입니다. 각 계층은 독립적인 역할을 수행하며, 이를 통해 문제 발생 시 어느 곳에서 문제가 생겼는지 원인 파악이 용이해집니다. 또한, 이 모델을 통해 각 계층에서의 작업을 논리적으로 분리할 수 있습니다. OSI 7계층은 다음과 같습니다.

OSI 7계층

계층이름
L7Application
L6Presentation
L5Session
L4Transport
L3Network
L2Data Link
L1Physical

이제 각 계층(Layer)에 맞는 프로토콜을 구현하여 네트워크를 구축할 수 있습니다.

OSI 7계층의 주요한 특징 중 하나는 각 레이어의 프로토콜이 하위 레이어의 프로토콜이 제공하는 기능을 사용하여 동작한다는 것입니다. 예를 들어, Network Layer에 구현된 프로토콜은 Data Link Layer에 구현된 프로토콜을 사용하여 자신의 기능을 구현하게 됩니다.

그리고 나중에 다시한번 다루겠지만 OSI 7계층은 어디까지나 참조 모델이고 실제 네트워크를 설계할때는 TCP/IP 모델을 사용합니다. TCP/IP 모델은 OSI 7계층을 4계층으로 나눈 모델입니다. 위 표에서 파란색으로 표시된 Allplication, Presentation, Session은 Application Layer로 통합되어 있습니다. 또한, Data Link와 Physical은 Network Access Layer로 통합되어 있습니다.

Allplication Layer

Application Layer는 사용자와 직접 상호작용하는 애플리케이션에 네트워크 서비스를 제공합니다. 이 계층은 HTTP, FTP, DNS, SMTP 등의 프로토콜을 포함하고 있습니다. 각 프로토콜이 어떠한 애플리케이션에 대응되는지 살펴보겠습니다.

  • HTTP: 웹 브라우저와 웹 서버 사이의 통신
  • FTP: 파일 전송
  • DNS: 도메인 이름 해석 (예: google.com → 172.217.161.142)
  • SMTP: 이메일 전송

Application Layer는 사용자가 사용하는 서비스를 제공하는 역할을 수행하며, 실제 데이터 전송 방식에 대해서는 하위 계층에 위임합니다.

Transport 레이어에서 Port 번호까지 봤으니 여기선 유투브를 보는지 줌회의를 하는지, 웹사이트를 보는지 같은 어떤 앱을 사용하는지 판단한다 생각하면 됩니다.

Presentation Layer

Presentation Layer는 애플리케이션 간의 통신에서 데이터를 어떻게 표현할지 정하는 역할을 수행합니다. 이를 메시지 포맷 관리라고도 합니다. 이 계층은 JPEG, MPEG, XDR (External Data Representation) 등의 데이터 포맷과 암호화 프로토콜을 포함하고 있습니다. 주요 역할은 다음과 같습니다.

  • 데이터 변환: Application Layer에서 읽을 수 있는 형식으로 데이터를 변환
  • 암호화, 복호화: 데이터를 암호화하여 전송하고, 수신측에서 복호화하여 데이터를 읽을 수 있게 함
  • 압축: 데이터를 압축하여 전송 효율을 높이고, 수신 측에서 압축 해제
  • 문자 인코딩: 다양한 문자 집합 간의 변환 (예: ASCII, Unicode)

Session Layer는 애플리케이션 간의 통신 세션을 관리하는 역할을 수행합니다. 세션이란 정보 교환을 전제하는 둘 이상의 통신장치나 컴퓨터와 사용자 간의 대화나 송수신 연결 상태를 의미합니다. 이 계층의 주요 기능은 다음과 같습니다.

  • 세션 설정, 유지 및 종료
  • 동기화 및 대화 제어
  • 데이터 교환 지점 설정 (체크포인트)

대표적인 프로토콜로는 NetBIOS, RPC (Remote Procedure Call), SSH (Secure Shell)가 있습니다. 참고로, TLS (Transport Layer Security)는 주로 **Transport Layer (L4)**에서 동작하지만, Session Layer의 기능도 일부 수행합니다.

이렇게 Application Layer, Presentation Layer, Session Layer를 알아보았습니다. 이러한 3개의 계층은 흔히 Application Layer로 통합되어 부르기도 합니다.

Transport Layer

Transport Layer(전송 계층)는 Application간의 통신을 담당합니다. 이 계층부터는 실제로 목적지 Application으로 데이터를 전송하는 일이 일어나게 됩니다. 추가적으로, OSI 7계층 모델에서는 하위 계층으로 갈수록 하드웨어에 가까워지기 때문에 하드웨어가 인지할 수 있는 데이터로 변환시켜줘야 합니다, 이때 Transport Layer는 하위 계층에 신뢰할 수 있는 데이터를 전송하는 역할도 수행합니다.

대표적으로 TCP (Transmission Control Protocol)와 UDP (User Datagram Protocol)가 있습니다.

  • TCP: 연결 지향적이며 신뢰성 있는 데이터 전송을 보장합니다.
  • UDP: 비연결 지향적이며 빠른 전송 속도를 제공합니다.

한가지 더 명심해야할 점은, Transport Layer는 어디까지나 Application끼리 어떻게 통신할지에 대한 약속을 정하는 역할을 합니다. 실제 데이터가 어떻게 목적지 까지 전달될지에 대한 역할은 하위 계층에 위임됩니다.

Network Layer

Application끼리 데이터를 주고 받으려면 결국 Host간의 통신이 먼저 이루어져야 합니다. 자연스럽게 Transport Layer 바로 밑에 있는 **Network Layer(네트워크 계층)**는 Host간의 통신을 담당합니다. 목적지 Host로 데이터를 전송하는 역할을 하기 때문에, Host 뿐만 아니라 Router또한 Network Layer기능을 포함하도록 설계되어 있습니다. 이 계층에는 IP, ICMP 등의 프로토콜이 포함됩니다.

  • IP: Host 간의 통신을 위한 주소 지정 체계를 제공합니다.
  • ICMP: 네트워크를 간단히 테스트하기 위한 프로토콜로, ping 명령어가 ICMP를 활용합니다.

Data Link Layer

Transport Layer에서 Application간의 통신을 위한 데이터를 만들어주었다면, Network Layer에서 Host간의 통신을 위한 데이터를 만들어주었습니다. 이제 **Data Link Layer(데이터 링크 계층)**에서는 직접 연결된 노드 간의 통신을 담당하게 됩니다. 노드와 노드사이의 신뢰성 있는 전송을 보장하며, IP 주소 기반이 아니라, MAC 주소를 통해 물리적인 장치를 식별하는 역할을 수행합니다.

  • ARP: IP 주소를 MAC 주소로 변환하는 프로토콜

Physical Layer

손으로 만질수 있는 물리적인 선을 의미합니다. 이 계층은 프로그래머의 역할이 아닌 전자공학과의 역할입니다.

이제 Internet Protocol(IP)에 대해 알아보겠습니다. IP는 Host간의 통신을 위해 사용되는 프로토콜로, 데이터를 패킷 단위로 전송합니다. IP는 패킷을 목적지까지 전달하기 위해 라우팅을 수행하며, 패킷이 목적지에 도착할 때까지 여러 노드를 거쳐 전달됩니다.

네트워크 계층은 IP 주소를 제공하는 역할을 수행한다.

수신자의 IP 주소를 기반으로 라우터가 경로를 결정해주게 된다. 이때의 경로는 라우터의 알고리즘에 의해 결정이 된다.

IP, ICMP, ARP 등의 프로토콜이 이 계층에 해당이 된다.

예를들어, ICMP 같은 경우는 네트워크를 간단히 테스트 하기 위한 ping 명령어가 ICMP를 활동을해 사용하는 방식이다.

ARP 같은 경우는 주변에 어떤 단말이 있는지 파악하기 위한 프로토콜이다.

전송 단위는 Packet 이다.

이전 시간에서 네트워크의 기본적인 개념을 알아보았습니다. 그렇다면 이제 이러한 네트워크를 만들때 어떠한 기능을 만들어야 할지 생각해 보겠습니다.

  1. 애플리케이션 목적에 맞는 통신 방법 제공
  2. 신뢰할 수 있는 데이터 전송 방법 제공
  3. 네트워크 간의 최적의 통신 경로 결정
  4. 목적지로 데이터 전송
  5. 노드 사이의 데이터 전송

OSI 7계층

국제표준화기구(ISO) 엣 개발한 모델로, 네트워크 통신을 단계별로 계층을 나눈 모델이다.

각 계층은 독립적인 역할을 부여 받아 동작하며, 이를 통해, 문제 발생시 어떤 현상인지 보고 어느곳에서 문제가 생겼는지, 원인 파악이 용이해진다.

또한 이 모델을 통해 각 계층에서의 작업을 시작적으로도 나눌 수 있다.

첫번째 열은 각 계층의 이름을 나타내며, 두번째 열은 프로토콜, 세번째 열언 PDU를 나타넨다.

데이터 전달 방식

러한 계층은 독립적으로 활동하고 있지만, 실제로는 서로 유기적으로 동작을 하고 있기 때문에, Stack 계층이라 표현 하기도 한다.

각 계층은 상위 계층에서 하위 계층으로 데이터를 **캡슐화(Encapsulation)**하여 전달하고 (7 -> 1),

반대로, 하위 계층에서 상위 계층으로 전달할때는 데이터를 **디캡슐화(De-Encapsulation)**하여 전달한다 (1 -> 7).

이를 쉽게 말해, 아래로 내려 갈수록, 보내고자 하는 데이터의 단위가 점점 커진다고 생각할 수있다.

Application Layer

HTTP, FTP, DHCP, POP3 등의 프로토콜이 이 계층이 포함이 된다.

우리가 일상적으로 사용하는 브라우저, 메일 시스템 등을 통해 해당 프로토콜을 용이하게 사용할 수 있게 해준다.

브라우저 같은것이 계층 자체를 의미하는게 아니라, 해당 소프트웨어로 인해 사용되는 프로토콜을 의미한다.

Presentation Layer

데이터를 어떻게 표현할지 정하는 역할을 가지고 있는 계층이다.

응용계층으로부터 받은 데이터를 읽을 수 있는 형식으로 변환하는 작업을 진행 한다.

이러한 변환은 암호화, 복호화, 압축, 문자 인코딩 등이 있다.

확장자의 개념이 여기서 사용이 된다.

Session Layer

응용 시스템 간의 세션을 관리 한다.

모든 통신 장치 간에 연결을 설정하고 관리 및 종료 하는 역할을 수행한다.

쉽게 말해, 멀티플레이 게임을 하기 위해 서버와의 통신을 할때 세션이라는 개념이 사용돼게 된다.

송신자와 수신자 간의 통신을 위해 동기화 신호를 주고 받는다.

통신 방식을 결정한다.(단순, 반이중, 전이중)

포트 연결이라고도 하며, SSH, TLS 등이 대표적인 프로토콜 이다.

Transpoty Layer

전송 계층의 주목적은 하위 계층으로 신뢰할 수 있는 데이터를 전송하는 것이다.

OSI7 계층 모델에서는 하위 계층으로 갈수록 하드웨어에 가까워 진다.

그래서 하드웨어가 인지할 수 있는 데이터를 만들어 주는 것이 전송 계층의 역할이다.

또한, End point 같 전달 데이터의 오류를 제어 하는 역할을 하기도 한다.(시퀀스 넘버 기반)

전송 단위는 Segment 이며, 대표적으로 TCP, UDP 프로토콜이 존재 한다.

TCP, UDP는 매우 중요한 개념으로 다른 시간에 알아보도록 하자, 간단히 요약하면,

TCP는 데이터의 오류를 방지해주는 역할이고, UDP는 데이터의 손실이 조금 있을수 있더라도 빠른 속도로 데이터를 전송하는 역할을 한다.

일반적으로 사용하는 프로토콜은 TCP 이고, UDP는 실시간 스트리밍, 알람 같은 곳에서 사용이 된다.

Network Layer

네트워크 계층은 IP 주소를 제공하는 역할을 수행한다.

수신자의 IP 주소를 기반으로 라우터가 경로를 결정해주게 된다. 이때의 경로는 라우터의 알고리즘에 의해 결정이 된다.

IP, ICMP, ARP 등의 프로토콜이 이 계층에 해당이 된다.

예를들어, ICMP 같은 경우는 네트워크를 간단히 테스트 하기 위한 ping 명령어가 ICMP를 활동을해 사용하는 방식이다.

ARP 같은 경우는 주변에 어떤 단말이 있는지 파악하기 위한 프로토콜이다.

전송 단위는 Packet 이다.

Data Link Layer

물리적인 네티워크를 통해 데이터를 전송하는 기능을 수행 한다.

주 목적은 물리적인 장치를 식별하는데 사용할 수 있는 주소 지정 체계를 제공하는 것이다.

대표적인 장비로는 브리지(Bridfe) 및 스위치(Switch)가 있다.

투 포인트 간의 신뢰성 있는 전송을 보장 한다.

여기서 투 포인트는 End point가 아닌 End point 사이에 있는 장비들 을 의미 하고, 여기에 사용되는 장비들이 스위치 다. 브리지는 요즘 잘 안쓰이는 추세이다.

사용 프로토콜은 MAC 이다.

참고로 MAC 은 통신을 하는 모든 장치들 모두 맥 주소를 보유 하고 있다.

Physical Layer

네트워크 데이터가 전송되는 물리적인 전송 기술을 의미하고, 0 또는 1 같은 비트 단위로 통신하게 된다.

이 계층에 속한 대표적인 장비는 케이블, 허브, 리피터 등이 있다.

여기서 말하는 허브 같은 경우는 사실 아무 기능을 하지 않고 그냥 데이터를 전송해주는 그런 역할을 하게 된다.

리피터 같은 경우는 이제 WAN 통신을 할때 사용이 되는데, 데이터를 전송할때 신호가 약해지는 것을 보완해주는 역할을 한다. 단순히 생각해, 데이터를 전송할때도 에너지가 필요할텐데, 이때 장거리 통신을 위해 에너지를 증폭시켜 주는 장비다.

케이블은 랜 케이블 같은것을 생각하면 된다.

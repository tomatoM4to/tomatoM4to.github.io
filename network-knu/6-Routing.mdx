# Routing algorithm

**global**: 나랑 인접한애 말고 나랑 떨어진 애도 모모든 그그래프에 대한 정보를 알고 있음
    1. link state algorithm
        * 보통 lan 환경에서 사용(작작은 환경)
            * 예를드어 IT대학, 경영대학 각각 lan 환경이 있다면, IT대학에서 경영대학으로 정보를 보낼때면 그냥 IT대학의 라라우터에게 전달하면 IT대학 라우터는 알아서 누군가에게 정보를 보냄, 외부에선 IT대학만 알고 IT대학 내내부에선 알아서 해결, 즉 크게크게 묶을수 있다는 의미다. 더 크게 보면 강원대 외부에선 강원대만 알면 돼고 강원대는 어느 대학인지만 할면 돼고 대학은 어느 학과인지, 학과는 누구인지 알면 돼는것처럼 계층적으로 구분 가능
            * 데이크스트라 씀

**Decenralized(분산 or 지역성과도 비슷한 의미)**:
    2. distance vector: 나랑 인접한애들한테 보내고, 인접한애는 또 인접한애들한테 보내고...
        * 보낼때 한명한테만 보내면 됀다, 알아야할 정보가 훨씬 적다, 하지만 인접한애가 새로운길을 찾았다면 다시 compute가 필요, 
        * 위 link state는 한번만 계산하면 돼지만 distance vector은 계산산이 많이 필요, 적은정보보하지만 많은 계산, 결국 수렴하는데 시간이 오래 걸림
        * 밸만 포드 알고리즘 씀

# 타입 검사 연산자
Dart에서는 `as`, `is`, `is!` 연산자를 통해 런타임에서 객체의 타입을 확인하거나 변환할 수 있습니다.

다른 언어의 `instanceof` 나 `type casting`과 비슷한 개념이라 익숙할 거예요.

| 연산자 | 의미 |
|---|---|
| `as` | 타입 캐스팅 (또는 라이브러리 prefix 지정 시 사용) |
| `is` | 객체가 해당 타입이면 true |
| `is!` | 객체가 해당 타입이 아니면 true |


```dart:타입
void main() {
    dynamic employee = Person();

    // is 연산자: 타입 검사
    if (employee is Person) {
        employee.firstName = 'Bob';
    }

    // as 연산자: 타입 캐스팅 (타입이 확실할 때만 사용)
    (employee as Person).firstName = 'Alice';
}
```

## 잠깐! 주의할 점
`is` 연산자는 검사만 하고, 타입을 바꾸진 않아요.

`as` 연산자는 타입을 강제로 변환하며, 타입이 다르면 예외를 발생시킵니다.

따라서 타입이 확실하지 않을 때는 `is`로 먼저 체크한 뒤 `as`를 사용하는 것이 안전한 패턴입니다.

## 참고
obj is Object?는 항상 true입니다. Dart에서는 모든 값이 Object?를 구현하기 때문입니다.

이 연산자들은 클래스, 인터페이스, 그리고 null 처리까지 다루는 부분에서 더 자주 사용됩니다. 자세한 건 클래스 파트에서 다시 설명할게요.

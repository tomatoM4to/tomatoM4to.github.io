# Lexical Closures (렉시컬 클로저)
이제 여기서 한발 더 나가서, **closure (클로저)**라는 개념이 나옵니다.

클로저란:
<R>스코프 안의 변수를 '잡아서' 기억하는 함수</R>를 말해요.
심지어 그 변수가 이미 스코프 밖으로 사라졌어도 말이죠!


예제: 클로저는 어떻게 변수를 기억할까?

```dart:closure
/// [addBy]만큼 더해주는 함수를 반환해주는 함수
Function makeAdder(int addBy) {
  return (int i) => addBy + i;
}

void main() {
  var add2 = makeAdder(2);   // 2를 더하는 함수 생성
  var add4 = makeAdder(4);   // 4를 더하는 함수 생성

  assert(add2(3) == 5);      // 2 + 3 = 5
  assert(add4(3) == 7);      // 4 + 3 = 7
}
```

여기서 중요한 건 `makeAdder` 함수 안의 `addBy` 변수예요.
이 변수는 `makeAdder` 가 끝나면 사라질 것 같지만,
`(int i) => addBy + i` 이 익명 함수가 `addBy` 를 잡아서 기억하고 있어요.

이게 바로 클로저입니다.

🧠 "내가 만들어질 때 주변에 뭐가 있었는지 기억하는 함수"

클로저는 언제 유용할까?
콜백 함수 안에서 외부 변수를 계속 기억하고 싶을 때

함수형 프로그래밍 스타일에서 **상태(state)**를 함수로 감싸고 싶을 때

간단한 팩토리 패턴을 함수로 구현할 때

이럴 때 아주 유용하게 쓰입니다.

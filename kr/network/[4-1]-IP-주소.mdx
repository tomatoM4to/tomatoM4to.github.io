Network LAN Card를 제조하는 업체해서 세팅하고 주는 주소로 6byte# Port, IP Address, MAC이 식별하는것
위 3개는 식별자라고 계속 표현했습니다. 그렇다면 과연 무엇을 식별할까요? 정답부터 표로 정리하면 다음과 같습니다.

| TCP/IP            | 식별자   | 식별 대상 |
|-------------------|-------------|--|
| Application Layer |             | |
| Transport Layer   | Port        | Process |
| Internet Layer    | IP Address  | Host |
| Link Layer        | MAC Address | NIC(Network Interface Card) |

IP Address와 Port번호가 무엇을 식별하는지는 이전부터 계속 말했기 때문에 넘어가겠습니다. 이제 처음나온 용어은 NIC에 대해 알아보겠습니다.

## NIC(Network Interface Card)
좀더 편하게 예기하면 그냥 LAN카드라고 이해해도 됩니다. 근데 이때 이렇게 생각해볼수 있는데요, 노트북 이용할때 유선으로도 쓸수 있지만 무선으로도 쓸수 있잖아요? 이게 가능한 이유는 유/무선 NIC아 여러개 있기 때문인데요, 이러한 NIC들에는 고유의 MAC Address가 1:1로 부여되어 있습니다. NIC은 Host에 부착된 그냥 부품에 불과한데요 이러한 MAC주소를 식별합니다.

# IPv4
일단 IP의 풀네임은 뭘까요? Internet Protocol이라고 합니다. 이 IP는 무엇을 식별할까요? Host를 식별합니다. 그럼 Host는 무엇일까요? Host는 무엇이든 될수 있습니다. 컴퓨터, 스마트폰, 서버, 라우터 등등.. 이러한 Host들은 IP Address를 가지고 있습니다. 이 IP Address에는 두가지의 버전이 있습니다. IPv4와 IPv6입니다. 이번에는 IPv4에 대해 알아보겠습니다.

IPv4와 IPv6의 차이는 주소의 길이입니다. IPv4는 32비트로 표현되며, IPv6는 128비트로 표현됩니다. 이러한 차이로 인해 IPv4는 2^32개의 주소를 가지고 있고, IPv6는 2^128개의 주소를 가지고 있습니다. 이 말은 세상에 존재하는 Host의 수가 한정되어 있다는 것입니다. IPv4기준 세상엔 2^32개의 Host가 존재할수 있습니다. 정말 작은숫자고, 중국 하나 조차 커버 못하는 숫자입니다. 이러한 IP 주소 부족 사태를 막기 위해 나온것이 IPv6 입니다.

하지만 현재 기준 IPv4를 보안하는 여러 기술들 때문에 IPv4를 사용하는 Host가 훨씬 많습니다. IPv6를 사용하는 경우는 매우 특별한 경우 말곤 없다고 봐도 됩니다. 따라서 IPv4 기준으로 설명하겠습니다.

지금 현재 IPv4 주소와 서브넷마스크, 게이트웨이가 있습니다. 먼저 주소 가지고만 예기 하겠습니다. IPv4주소는 4개의 8비트로 이루어져 있습니다. 이 4개의 8비트를 10진수로 변환하면 이 8비트의 수는 `.`을 기준으로 나누어져 있습니다. 그러므로 IPv4주소는 `[0~255].[0~255].[0~255].[0~255]`로 이루어져 있습니다. 이것이 IPv4의 구조입니다.

그리고 이제 서브넷 마스크에 대해 알아보겠습니다. 사실 IPv4 주소는 두 부분으로 다시 나뉘어 집니다. 이 나누는것을 서브넷 마스크로 하게 돼는데요. 어떻게 하는지 알아보겠습니다.

먼저 IP주소를 2진수로 바꾸고, 서브넷 마스크도 2진수로 바꿉니다. 그리고 국 부분을 AND 연산을 하게 됍니다. 이렇게 돼서 나뉘어진 두 부분을 각각 NETWORK ID, HostID로 나눕니다.

혹은 192.168.60.14/24 이렇게 나누어서 앞 24bit는 NETID, 그 뒤는 HostID 이렇게 표기 하기도 합니다.

## IP 주소의 종류
IP 주소 구조를 2^8로 나누어서 4개의 8비트로 나누어서 표현한다 했습니다. 그리고 다시  Network ID 와 Host ID로 나누어서 표현한다 했습니다. 이제 IP 주소의 종류에 대해 알아보겠습니다.

IP 주소는 크게 4가지로 나뉘어 집니다.
1. Global IP
2. Private IP
3. Loopback IP
5. Broadcast IP

### Global IP
GlobalIP 또는 Public IP라고도 불립니다. 일단 인터넷이란 무엇인가 에 대해서 다시 한번 생각해봐야 합니다. 인터넷이란 수많은 호스트들의 연결이고 이러한 Host들간의 데이터는 Router로 Switching 돼어진다 했는데요, 이때 이 Router가 Switching을 하는 대전제의 조건이 있습니다. 그것은 목적지의 주소를 보고 GlobalIP일때만 라우팅을 합니다. 그러므로 PrivateIP는 기본적으로 라우팅을 해주지 않습니다. 특징점 중 하나는 이러한 GlobalIP는 고유한 주소를 가지고 있습니다.

### Private IP
말 그대로 사설인데 작은 소규모 인터넷을 구축하기 위해 사용돼는 IP입니다. GlobalIP로 이루어진 진짜 인터넷하고 PrivateIP로 이루어진 작은 인터넷의 원리적 차이는 없습니다. 또 특징점중 하나는 이러한 PrivateIP는 고유한 주소를 가지고 있지 않습니다.

#### IP주소의 Class
IP주소는 Class로 나뉘어 집니다. Class는 5개로 나뉘어 집니다. A, B, C, D로 나뉘어 집니다. 이러한 Class는 Network ID와 Host ID의 비트수로 나뉘어 집니다. 이러한 Class는 다음과 같습니다.

1. Class A: 8bit Network ID, 24bit Host ID
    * Class A 네트워크란 의미는 해당 네트워크 안에 Host가 2^24개 있을수 있다는 뜻입니다. 초 거대 네트워크입니다.
2. Class B: 16bit Network ID, 16bit Host ID로
    * 보통 큰 대학교 정도가 가지고 있습니다.
3. Class C: 24bit Network ID, 8bit Host ID
    * 보통 가정에서 사용합니다.

이러한 Class별로 Private IP 주소가 정해져 있습니다. 그리고 이러한 PrivateIP 주소는 공유기에서 매우 자주 사용합니다.
* Aclass: 10.x.x.x
* Bclass: 172.16.x.x
* Cclass: 192.168.x.x

##### 공유기의 역할
일반적으로 인터넷을 사용하기 위해선 GlobalIP가 필요합니다. 하지만 그냥 사용하면 IP주소 하나에 Host 하나가 할당되는데요, 이러한 방식은 비효율적입니다. 그래서 공유기를 사용합니다. 공유기는 하나의 GlobalIP를 가지고 있고, 이 GlobalIP를 가지고 있는 공유기는 여러 Host들에게 PrivateIP를 할당합니다. 이러한 방식으로 하나의 GlobalIP로 여러 Host들이 번갈아가며 사용해 마치 여러 Host들이 GlobalIP를 가지고 있는것처럼 보이게 합니다.

### Loopback IP
127.0.0.1 형태의 주소입니다. 보통 웹개발시 자주 사용합니다. 이 주소는 Host 자신을 가리키는 주소입니다. 이 주소로 데이터를 보내면 Host는 데이터를 받아들이고 다시 보내는것이 아니라 그냥 받아들이고 버립니다. 이 주소는 Host가 정상적으로 작동하는지 확인하기 위해 사용됩니다.

# 서브넷팅
모든 지식을 배울때 가장 중요한것은 왜 이게 필요하지? 부터 시작해야 한다 생각합니다.


IP주소의 구조에 대해 알면 아주 쉽게 서브넷팅을 이해할수 있습니다. 서브넷팅은 IP주소를 나누는것입니다. 이것은 Network ID와 Host ID를 나누는것입니다. 이것은 서브넷 마스크를 이용해서 나누어 집니다.

# 무선 Network를 이루는 장치들

## 무선 호스트
유선 네트워크의 경우처럼 호스트는 애플레케이션을 실행하는 종단 시스템 장치로서 스마트폰, 태블릿, 랩톱, 센서 같은 IoT(Internet of Things) 장치를 포함한다.

## 무선 링크
**무선 링크(Wireless Link)**는 데이터를 전달하기 위해 물리적인 케이블 대신 무선 신호를 사용하는 통신 경로를 의미합니다.

주요 예로는 <R>4G, 5G, Wi-Fi, 블루투스, 위성 통신</R> 등이 있습니다.

호스트는 무선 통신 링크(wireless communication link)를 통해 기지국(base station) 이나 다른 무선 호스트에 연결된다.

여러 무선 링크 기술들은 전송 속도와 전송 거리 면어세 차이가 존재한다.

무선 링크는 때로 네트워크 코어에서도 라우터, 스위치 등 네티워크 장치들 사이를 연결해 주는데 사용될 수 있다. 하지만 대부분 호스트를 기준으로 발전해왔기 때문에 호스트를 기준으로 설명한다.

## 기지국(Base Station)
무선 네트워크 인프라스트럭처의 핵심 부분

<R>무선 네트워크에서는 데이터를 송수신하기 위해 기지국(Base Station)이 필요하다.</R>

유선 네트워크에서는 데이터가 물리적 케이블과 네트워크 장치(예: 스위치, 라우터)를 통해 직접 전달되기 때문에, 무선 네트워크의 기지국(Base Station)처럼 데이터를 중재하는 특별한 장치가 필요하지 않다.

Base station은 Base station에 결합(association)된 무선 호스트와의 데이터(패킷) 송수신에 대한 책임을 진다. Base station은 자신과 결합되어 있는 다수의 무선 호스트들 사이의 전송을 중재하고 조정하는 역할도 한다.

기지국의 종류론 크게 2가지로 나뉜다.
1. cell tower: 휴대전화 네트워크에서 사용되는 기지국
2. AP(Access Point): Wi-Fi 네트워크에서 사용되는 기지국, 공유기가 대표적인 예

### 결합(association)
결합(association)이란 무선 호스트가 기지국과 연결되어 통신할 준비가 된 상태를 말하며, 기지국의 무선 통신 영역 안에 있고, 호스트가 데이터 전송을 위한 중계 장치로 해당 기지국을 선택해야 합니다.

핸드오프(handoff)는 무선 호스트가 한 기지국에서 다른 기지국으로 이동하는 것을 의미합니다.

호스트가 한 기지국 영역을 벗어나 다른 기지국 영역으로 이동하는 경우, 중심부에 있는 Base station을 변경해야 하며, 이 과정을 핸드오프(hand-off) 라고 합니다.

핸드오프에서 다루어야 할 기술적 도전들 소개
1. 어떻게 네트워크 내에서 이동 호스트의 현재 위치를 알아내어 그 호스트로 데이터를 전달할수 있을까?
2. 지점을 특정할수 있다면, 어떻게 적합한 주소를 할당할 수 있을까?
3. 호스트가 TCP 연결이나 전화통화 도중에 이동하는 경우, 연결이 끊어지지 않고 지속될 수 있도록 데이터를 어떻게 전달해야 할까?
4. ... 등

매우 도전적이면서 어려운 문제들이 많이 존재한다.

## 무선 네트워크의 분류
무선 네트워크는 인프라스트럭처의 유무와 홉(hop)의 수에 따라 크게 네 가지로 분류할 수 있습니다.

인프라스트럭처:

### 단일 홉, 인프라스트럭처 존재
설명: 네트워크에 **인프라스트럭처(기지국, AP 등)**가 존재하며, 모든 통신은 단일 홉(single hop)을 통해 이루어집니다. 즉, 무선 장치(호스트)는 직접 인프라스트럭처와만 통신합니다.

예시:
* Wi-Fi 네트워크에서 단말기(노트북, 스마트폰)가 **AP(Access Point)**와 직접 연결되는 경우.
* 셀룰러 네트워크(4G/5G)에서 스마트폰이 기지국과 통신하는 경우.


### 단일 홉, 인프라스트럭처 없음
설명: 네트워크에 인프라스트럭처가 존재하지 않으며, 모든 통신은 단일 홉(single hop)을 통해 이루어집니다. 무선 장치들이 서로 직접 통신합니다.

예시:
* 블루투스 네트워크(페어링된 두 장치 간 직접 통신).
* Wi-Fi Direct 또는 애드혹(ad-hoc) 네트워크에서 단말기들이 서로 직접 연결된 경우.

### 다중 홉, 인프라스트럭처 존재
설명: 네트워크에 인프라스트럭처가 존재하지만, 통신이 다중 홉(multi-hop)을 통해 이루어집니다. 무선 장치들은 중간 장치를 거쳐 인프라스트럭처와 연결됩니다.

예시:
* 메시 네트워크(Mesh Network)에서 일부 노드가 AP나 기지국에 연결되고, 다른 노드들이 이를 통해 간접적으로 통신하는 경우.
* 위성 통신 네트워크에서 중계 위성을 통해 다중 홉 경로로 데이터를 전송하는 경우.

### 다중 홉, 인프라스트럭처 없음
설명: 네트워크에 인프라스트럭처가 존재하지 않으며, 통신이 다중 홉(multi-hop)을 통해 이루어집니다. 무선 장치들이 중계 노드 역할을 하여 데이터를 전송합니다.

예시:
* 모바일 애드혹 네트워크(MANET: Mobile Ad-hoc Network).
* 재난 상황에서 사용하는 무선 센서 네트워크(WSN: Wireless Sensor Network).

# 무선 링크와 네트워크의 특징
무선 링크는 유선 링크와 비교하여 여러가지 차이점이 존재

* 신호 세기의 감소: 전자기파는 물체를 통과함에 따라 약화됨, 자유 공간에서도 전자기파 신호는 분산되고 거리에 따라 신호 세기가 감소, 이런 현상을 경로 손실(path loss)이라고 함.

* 다른 출발지로부터의 간섭: 동일 주파스 대역으로 전송되는 무선 신호들은 서로 간섭하게 됨, 예를들어 **2.4 GHz 대역의 무선전화**와 **802.11b 무선 랜(Wi-Fi)**은 동일한 주파수 대역을 사용하기 때문에 동시에 사용하는 경우, 서로 제대로된 성능이 나오기 기대하기 어려움, 근처의 모터나 전자레인지도 간섭을 일으킬수 있음
    * 이런 이유로 최근 **802.11b 무선 랜(Wi-Fi)** 표준들은 은 5GHz 개역으로 동작

* 다중 경로 전파: 송신자와 수신자 간에 전송되는 전자기파들이 물체나 지표에 부딪혀 서로 다른 경로를 통해 수신자에 도달할 수 있음, 이런 현상을 다중 경로 전파(multipath propagation)이라고 함, 이는 신호의 강도와 위상을 변화시키는 다양한 문제를 일으킬 수 있음

좀더 자세한 무선 채널의 특성, 모델, 측정에 대해선, [Anderson 1995, Almers 2007]을 참고

이러한 특징때문에 무선 링크는 비트 오류가 자주 발생할 수 있음

이로인해 강력한 CRC 오류 검출 코드를 사용할 뿐만 아니라, 손상된 프레임을 재전송 해주는 링크 레벨의 신뢰성 있는 데이터 전송 프로토콜 사용

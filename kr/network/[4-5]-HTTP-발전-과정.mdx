# HTTP/0.9

현재는 거의 사용되지 않는 초창기 HTTP 버전

사용 가능한 메서드는 GET 하나뿐

헤더또한 없고, 단순히 요청과 응답의 Body 만 포함됨

# HTTP/1.0

HEAD, POST 등 다양한 메서드를 지원

헤더 를 통해 클라이언트와 서버 간의 추가 정보를 교환할 수 있음

공식적으론 지속 연결을 미지원

# HTTP/1.1
지속 연결을 기본적으로 지원

이 버전까진 Body 가 평문으로 되어있어 인간이 읽기 쉬운 형태

## HOL 블로킹 문제

`HTTP/1.1` 에서 지속 연결을 지원하지만, 하나의 TCP 연결에서 여러 개의 요청을 처리할 때 발생하는 문제

간단히 말해 `요청A` 가 완료되기 전까지 `요청B` Queue 에 대기해야 하는 상황

# HTTP/2

구글의 `SPDY` 프로토콜을 기반으로 개발된 HTTP 의 차세대 버전

간단히 `HTTP/1.1` 의 효율과 성능을 높이기 위한 버전임

Body 가 바이너리 데이터 형태인것 또한 특징

Header 를 압축하는 기능 또한 지원

Server push 기능을 통해 클라이언트가 요청하지 않은 리소스도 서버가 미리 전송할 수 있음

해당 버전은 HTTP multiplexing 을 지원하여, 하나의 TCP 연결에서 여러 개의 요청을 stream 이라는 단위를 활용해 병렬적으로 처리할 수 있음

Request-Response 단위는 하나의 Stream 에서 이루어 짐

Stream 별 독립적인 흐름을 유지 가능

결과적으로 HOL 블로킹 문제를 상당부분 해결함

# HTTP/3
가장 최신의 HTTP 버전

이전 버전까진 기본적으로 TCP 기반이었지만, HTTP/3 는 UDP 기반 프로토콜인 QUIC 프로토콜을 기반으로 함

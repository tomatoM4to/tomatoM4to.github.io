이번 주제에서는 **실습 중심**으로 데이터베이스의 기본 개념을 익히고, 실제로 설계하고 관리해보는 과정을 함께해 볼 겁니다.

이론은 꼭 필요한 부분만 다루고, 대부분은 직접 해보면서 배우는 방식으로 구성했어요. 우리가 실습에 사용할 데이터베이스는 **PostgreSQL** 입니다.

물론, 이 한 권으로 데이터베이스의 모든 걸 다 배우긴 어렵습니다. 데이터베이스는 정말 방대한 분야니까요. 하지만 이 글을 통해 기초적인 개념을 익히고, 실습을 통해 데이터베이스가 어떻게 작동하는지 감을 잡는 것을 목표로 해요.

# 데이터베이스란 무엇인가?

먼저, `Data` 라는 게 뭔지부터 알아볼게요.

* 데이터: 어떤 사실이나 현상을 관찰해서 얻은 수치나 기록 같은 것
* 정보: 데이터에 의미를 부여한 것
* 지식: 정보가 쌓여서 생긴 이해나 통찰

이걸 바탕으로 데이터베이스를 정의해보면, **정보나 지식을 뽑아내기 위해 데이터를 잘 모아놓은 Software** 라고 말할 수 있을거 같아요.

사실 `데이터베이스` 라는 말은 조금 추상적인 단어 일 수도 있을거 같아요. 엄밀하게 이게 바로 **데이터베이스다!** 하고 딱 잘라 정의하긴 어렵거든요.

하지만 우리가 데이터베이스를 공부한다는 건, **결국 값(데이터)을 저장하고, 찾아보고, 수정하는 일을 좀 더 안전하고 효율적으로 해주는 Software**를 배우는 거라고 생각하면 돼요.


## 역할
데이터베이스의 역할은 크게 2가지로 나눌수 있습니다.

1. Read: 검색
2. Write: 변경

데이터베이스의 가장 큰 목적인 **수많은 데이터를 빠르고, 안전하게 저장, 검색, 수정** 하는 것입니다.

우리가 매일 쓰는 앱이나 웹사이트도 대부분 이 두 가지 기능을 기반으로 동작하죠.


## 데이터베이스의 특징

데이터를 데이터베이스에 저장할 때는 중복을 최대한 줄이고, **여러 사람이 함께** 사용할 수 있도록 잘 정리해서 저장해야 해요.

조금 더 구체적으로 보면, 데이터베이스에 저장되는 데이터는 다음과 같은 규칙을 따라야 해요. 안지켜도 동작은 하겠지만 지키는게 좋겠죠.

데이터베이스에 데이터를 저장할 때는 중복을 최소화하며, 특정한 목적과 함께 여러 사람이 공유할 수 있도록 저장해야 합니다.


자세한 특징은 다음과 같습니다. 하지만 이러한 특징들 중, 2번과 3번 특징은 별로 의미 있는 특징은 아니라 할 수 있겠습니다, 너무나 당연하게 컴퓨터에 저장하고, 쓸모없는 데이터를 저장할 일은 별로 없기 때문이죠.

1. 통합된 데이터(Integrated Data)
    * 중복을 최소화하고, 서로 연결된 데이터 간의 일관성을 유지한 상태 여야 합니다.
2. 저장된 데이터(Stored Data)
    * 컴퓨터가 읽고 쓸 수 있는 저장 장치에 저장된 형태의 데이터를 말합니다.
3. 운영 데이터(Operational Data)
    * 실제로 업무에 사용되는 데이터를 말해요.
4. 공용 데이터(Shared Data)
    * 한 사람이 아닌, 여러 사람이 함께 사용하는 데이터를 뜻해요.

사실 2번과 3번은 너무 당연한 내용이라, 굳이 외우려고 애쓸 필요는 없어요. 데이터는 당연히 저장되어야 하고, 당연히 필요한 정보만 저장하겠죠?

그렇다면 **데이터베이스 자체는 어떤 특징**을 가질까요? 앞에서 이야기한 데이터의 성격과 맞물려, 데이터베이스 시스템은 다음과 같은 성격을 가져요.

1. 실시간 접근성(Real-time Access)
    * 사용자가 데이터를 요청하면, 거의 즉시 응답할 수 있어야 해요.
2. 계속적인 변화(Continuous Evolution)
    * 새로운 데이터를 추가하고, 기존 데이터를 수정하거나 삭제할 수 있어야 해요.
3. 동시 공유성(Concurrent Sharing)
    * 여러 사용자가 동시에 데이터베이스에 접속해서 작업할 수 있어야 해요.
4. 내용 참조(Content Reference)
    * 데이터를 찾을 때, 물리적인 위치가 아니라 **내용(값)**을 기준으로 참조해요.


## SQL
SQL은 Structured Query Language의 줄임말로, **데이터베이스를 관리하고 다루기 위한 선언형 언어**예요.

선언형 언어 라는 점 때문에 일반적인 프로그래밍 언어처럼 생각하면 조금 헷갈릴 수 있을겁니다. C 나 Python 같은 언어는 loop 나 if 같은 요소를 사용해 컴퓨터에게 정확히 무엇을 해야 할지 지시한다면, 선언형 언어인 SQL 은 시스템에게 **무엇을 해야 하는지**를 요청하는 방식으로 동작합니다.

지금 이 말이 와닫지 않더라도 괜찮습니다. 이후 SQL 을 다룰때 이게 무슨 말인지 조금씩 이해하게 될거예요.

```SQL
SELECT name, age
FROM users
```

위 코드는 `users` 라는 테이블에서 `name` 과 `age` 라는 두 가지 정보를 조회하는 SQL문이에요.

혹시 이 코드가 아직 잘 이해되지 않아도 전혀 걱정하지 마세요! 3장부터 SQL문법을 하나씩 차근차근 배울 예정이니까요.

# 데이터베이스 시스템의 구성

데이터베이스 시스템은 보통 **일반 사용자가 직접 접근하지 않도록 설계**돼요. 뿐만 아니라 시스템을 설계하는 개발자나 관리자(DBA) 역시 데이터베이스 자체에 직접 접근하지 않고, **중간에서 관리해주는 소프트웨어(DBMS)**를 통해 작업하죠.

이 단락에서는 데이터베이스를 운영하는 사람과, 그것을 관리해주는 시스템에 대해 살펴볼게요.

1. 데이터베이스 관리자 (DBA)
    * 데이터베이스를 설계하고 구축하며, 전체 데이터베이스 시스템의 운영과 유지보수에 대한 책임을 집니다.
    * 보안, 백업, 성능 튜닝 등 전문적인 관리 작업을 담당하는 **사람**입니다.
2. DBMS (Database Management System)
    * **데이터베이스를 실제로 관리해주는 소프트웨어**

## DBMS (Database Management System)

DBMS는 데이터베이스를 생성하고, 데이터를 **저장·조회·수정·삭제**할 수 있도록 도와주는 **소프트웨어**예요. 즉, 개발자나 DBA가 물리적인 실제 데이터에 직접 접근하지 않고 DBMS 를 통해 데이터베이스를 관리해줄 수 있도록 해주는 거죠.

또 DBMS는 앞서 설명한 실시간 접근성, 계속적인 변화, 동시 공유성, 내용 참조 같은 기능들을 기본적으로 제공하면서, 데이터베이스의 **백업과 복구·보안·성능 튜닝·병행 제어·무결성** 같은 보다 전문적인 기능까지 제공합니다.

결국 DBA 가 된다는 건, DBMS를 잘 다룰 줄 아는 사람이 된다는 의미이기도 해요.

실제로 우리가 흔히 들어본 Oracle, MySQL, PostgreSQL 같은 것들이 모두 DBMS입니다. 이 책에서는 **PostgreSQL** 을 사용해서 실습을 진행할 거예요.

다음으로 DBMS의 기능에 대해 간략히 소개하고 넘어가겠습니다.

1. 데이터 **정의**: 데이터의 구조를 정의하고, 필요 시 변경하거나 삭제하는 기능
2. 데이터 **조작**: 데이터를 조회, 추가, 수정, 삭제하는 기능
3. 데이터 **추출**: 데이터베이스에서 필요한 데이터를 외부로 꺼내는 기능 (예: 웹사이트로 보내기)
4. 데이터 **제어**: 접근 권한 설정, 백업 및 복원, 동시에 여러 사용자가 작업할 때 충돌을 방지하는 제어 기능 등

## DBMS와 File System

1950-60년대, 컴퓨터가 막 보급되던 시절에는 데이터를 저장할 때 파일 시스템을 사용했어요. 우리가 흔히 아는 방식인 **폴더에 파일을 넣고 정리하는 방식**이죠.

지금도 간단한 데이터는 파일 시스템으로 충분히 저장할 수 있어요. 하지만 데이터가 수십만 개, 수천만 개, 그 이상으로 많아지면서 문제가 생기기 시작했습니다.
1. 데이터 검색 속도가 느려짐
2. 데이터 중복이나 손실 관리가 어려워짐
3. 여러 사람이 동시에 데이터를 수정할 때 충돌이 발생함

**이런 문제를 해결하기 위해 DBMS가 등장**했어요.

그렇다고 파일 시스템이 무조건 나쁘다는 건 아니에요! 작고 단순한 데이터라면 오히려 파일 시스템이 더 가볍고 빠를 수도 있어요. **DBMS는** CPU, 메모리 등 리소스를 많이 쓰기 때문에 **데이터가 정말 많을 때만 그 진가를 발휘**합니다.

정리해서, DBMS 는

> **이제 사람이 직접 관리하긴 너무 많고 복잡한 데이터**를 체계적으로, 빠르게, 그리고 안전하게 다루기 위한 무거운 소프트웨어

라고 정의할 수 있을거 같습니다.
